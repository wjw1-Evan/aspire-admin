# 自动构建命令

## 描述

编译整个解决方案的后端代码，并构建 Admin 前端项目。

## 执行顺序

1. 后端构建
2. 前端构建

## 步骤

### 1. 后端构建

在项目根目录执行 dotnet build：

```bash
dotnet build --no-restore
```

- `--no-restore`：跳过恢复步骤，直接使用已缓存的依赖
- 允许警告，阻止编译错误
- 构建成功后再执行下一步

### 2. 前端构建

进入 Platform.Admin 目录执行：

```bash
cd Platform.Admin
npm run lint
npm run build
```

- `lint`：运行类型检查和代码规范检查
- `build`：执行生产构建

## 规则

- **顺序执行**：先完成后端构建，再执行前端构建
- **错误处理**：如有构建错误，立即报告并终止，不执行后续步骤
- **警告容忍**：警告不影响构建流程继续执行
- **构建结果**：完成后报告构建状态

## 错误报告

如有错误，提供以下信息：
- 出错的文件路径和行号
- 错误类型（编译错误/类型错误/lint 错误）
- 简要描述和可能的解决方案
