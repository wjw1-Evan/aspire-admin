# 多分支条件节点使用指南

## 快速开始

### 1. 创建条件节点
在工作流设计器中添加一个条件节点。

### 2. 配置分支

#### 添加分支
1. 打开条件节点配置抽屉
2. 在"业务规则"标签页中找到"条件分支"部分
3. 点击"添加分支"按钮

#### 配置单个分支
每个分支包含以下配置项：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 分支标签 | 分支的描述性名称 | "金额 > 1000" |
| 启用此分支 | 是否启用该分支 | ✓ 启用 |
| 条件规则 | 分支匹配的条件 | 见下表 |
| 条件间逻辑 | 多个条件的组合方式 | AND / OR |
| 目标节点 | 分支匹配时的下一个节点 | 选择审批节点 |

#### 配置条件规则
每个分支可以包含多个条件规则：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 表单 | 条件涉及的表单 | 选择"申请表" |
| 字段 | 表单中的字段 | 选择"金额" |
| 操作符 | 比较操作符 | 大于、小于、等于等 |
| 比较值 | 与字段值比较的值 | 1000 |

#### 设置默认分支
1. 在"默认分支"下拉框中选择一个分支
2. 当所有条件都不匹配时，流程将进入默认分支

### 3. 连接分支到下一个节点
- 每个分支对应条件节点底部的一个输出 handle
- 从分支 handle 拖拽连线到下一个节点

## 常见场景

### 场景 1：简单的二分支判断
**需求**：根据申请金额判断是否需要高级审批

**配置**：
- 分支 1：金额 > 10000 → 高级审批
- 分支 2：金额 ≤ 10000 → 普通审批
- 默认分支：分支 2

**条件规则**：
```
分支 1：
  - 表单：申请表
  - 字段：金额
  - 操作符：大于
  - 比较值：10000
  - 目标节点：高级审批

分支 2：
  - 表单：申请表
  - 字段：金额
  - 操作符：小于等于
  - 比较值：10000
  - 目标节点：普通审批
```

### 场景 2：多条件组合判断
**需求**：根据申请类型和金额判断审批流程

**配置**：
- 分支 1：类型=采购 AND 金额>50000 → 采购审批
- 分支 2：类型=差旅 AND 金额>5000 → 差旅审批
- 分支 3：其他 → 普通审批

**条件规则**：
```
分支 1（条件间逻辑：AND）：
  - 条件 1：申请表.申请类型 == "采购"
  - 条件 2：申请表.金额 > 50000
  - 目标节点：采购审批

分支 2（条件间逻辑：AND）：
  - 条件 1：申请表.申请类型 == "差旅"
  - 条件 2：申请表.金额 > 5000
  - 目标节点：差旅审批

分支 3（无条件）：
  - 目标节点：普通审批
```

### 场景 3：多选项判断
**需求**：根据申请类型路由到不同的审批流程

**配置**：
- 分支 1：类型 = 采购 → 采购部审批
- 分支 2：类型 = 差旅 → 行政部审批
- 分支 3：类型 = 其他 → 总经理审批

**条件规则**：
```
分支 1：
  - 表单：申请表
  - 字段：申请类型
  - 操作符：等于
  - 比较值：采购
  - 目标节点：采购部审批

分支 2：
  - 表单：申请表
  - 字段：申请类型
  - 操作符：等于
  - 比较值：差旅
  - 目标节点：行政部审批

分支 3（无条件）：
  - 目标节点：总经理审批
```

## 操作符说明

| 操作符 | 说明 | 示例 |
|--------|------|------|
| 等于 (==) | 字段值等于比较值 | 金额 == 1000 |
| 不等于 (!=) | 字段值不等于比较值 | 状态 != "已取消" |
| 大于 (>) | 字段值大于比较值 | 金额 > 1000 |
| 小于 (<) | 字段值小于比较值 | 金额 < 1000 |
| 大于等于 (>=) | 字段值大于等于比较值 | 金额 >= 1000 |
| 小于等于 (<=) | 字段值小于等于比较值 | 金额 <= 1000 |
| 包含 (Contains) | 字段值包含比较值 | 备注 包含 "紧急" |

## 最佳实践

### 1. 分支顺序
- 将更具体的条件放在前面
- 将通用条件放在后面
- 系统按顺序评估分支，返回第一个匹配的分支

### 2. 默认分支
- 始终设置一个默认分支
- 默认分支应该处理所有未被其他分支覆盖的情况

### 3. 条件设计
- 避免分支条件重叠
- 确保所有可能的情况都被覆盖
- 使用清晰的分支标签便于理解

### 4. 性能考虑
- 避免过多的分支（建议不超过 10 个）
- 复杂的条件逻辑可以使用自定义表达式

## 故障排除

### 问题 1：流程没有进入预期的分支
**可能原因**：
- 条件规则配置错误
- 表单数据不符合条件
- 分支顺序不对

**解决方案**：
1. 检查条件规则中的字段名和比较值
2. 验证表单数据是否正确填充
3. 调整分支顺序，确保更具体的条件在前面

### 问题 2：流程进入了默认分支
**可能原因**：
- 所有分支条件都不匹配
- 分支被禁用

**解决方案**：
1. 检查分支是否启用
2. 验证条件规则是否覆盖了所有情况
3. 查看流程日志了解条件评估结果

### 问题 3：条件节点显示"等待配置条件分支"
**可能原因**：
- 没有添加任何分支

**解决方案**：
1. 打开条件节点配置
2. 点击"添加分支"按钮
3. 配置至少一个分支

## 高级用法

### 自定义表达式
如果结构化规则无法满足需求，可以使用自定义 C# 表达式：

```csharp
// 示例：复杂的业务逻辑
{amount} > 1000 && {type} == "采购" || {priority} == "高"
```

### 变量引用
在条件中可以引用以下变量：
- 表单字段：`{fieldName}`
- 流程变量：`{variableName}`
- 系统变量：`{__instanceId}`, `{__documentTitle}` 等
