# 微信风格视觉与交互基线

## 🎯 目标

- 让 Platform.App 的视觉体验贴合微信最新版本（iOS / Android）的界面语言。
- 提供统一的主题、组件、交互规范，作为研发和测试的基线。
- 在满足微信风格的同时保留 Aspire 品牌标识（如必要，可在启动页、图标等位置注入品牌元素）。

## 🎨 视觉规范

### 1. 配色体系
- **主色（Brand / Accent）**：`#07C160`（微信绿色），用于按钮、强调图标、链接等。
- **辅色**：
  - 强调橙色：`#FA9D3B`（用于状态提示、徽章）
  - 成功绿：`#07C160`
  - 警告黄：`#FFB300`
  - 错误红：`#F25C54`
- **背景**：
  - 主要背景：`#F4F4F4`
  - 卡片 / 列表白底：`#FFFFFF`
  - 深色模式主要背景：`#111111`
- **文本颜色**：
  - 一级文本：`#000000` / 深色模式 `#FFFFFF`
  - 次级文本：`#888888` / 深色模式 `#777777`
  - 禁用文本：`#C0C0C0`
- **分隔线 / 边框**：`#E5E5E5`

### 2. 字体与字号
- 使用系统字体（iOS：SF Pro；Android：Roboto/Noto Sans）。
- 主要字号：
  - 标题 / 主导航：18px / 20px、字重 `600`。
  - 列表标题：16px、字重 `500`。
  - 普通文本：14px、字重 `400`。
  - 辅助说明：12px、字重 `400`。
- 字间距保持系统默认，不额外设置 letter-spacing。

### 3. 圆角与阴影
- 按钮、卡片、输入框：圆角 8px。
- 对话气泡：上下 8px，左右 12px。
- 阴影仅在需要浮起效果时使用，透明度低，参考微信（iOS：`shadowOffset={width:0,height:2}`，`shadowOpacity=0.1`）。

### 4. 间距与布局
- 页面左右边距：16px。
- 列表项上下 padding：12–16px。
- 卡片内部：上下 16px，左右 16px，元素之间垂直间距 8–12px。
- 分组之间使用 16px 间距 + 灰色背景区分。

## 🧱 组件规范

| 组件 | 样式要求 | 交互要求 |
| ---- | -------- | -------- |
| 顶部导航 | 白底 + 细线分隔，标题居中，返回按钮为箭头 | 支持滑动返回，标题渐隐/显 |
| 底部 Tab | iOS 风格图标 + 文本，激活色为微信绿 | 切换时无动画或轻微淡入 |
| 列表单元 | 左图标 + 主/副标题 + 右箭头，整行可点击 | 点击态背景 `rgba(0,0,0,0.05)` |
| 按钮 | 实心圆角 8px，主色为微信绿，文字白色 | 禁用时降低透明度 |
| 输入框 | 灰色圆角矩形，内部 padding 12px | 激活时边框变绿，光标颜色与主色一致 |
| 对话气泡 | 圆角矩形，左侧灰色右侧绿色，字体 16px | 支持长按弹出操作菜单 |
| 弹窗 / ActionSheet | 白底圆角 12px，按钮居中排列 | 弹出/消失使用淡入淡出 + 缩放 |
| Toast | 居中深色半透明背景，文字白色，圆角 8px | 自动消失，时长 2 秒 |

## ♻️ 交互与动效

- **返回手势**：保持 iOS 左滑返回、Android 物理返回键行为。
- **下拉刷新**：使用系统原生 `RefreshControl`，样式为微信常用的下拉段落。
- **加载状态**：顶部展示微信式加载条或轻量 Loading spinner。
- **页面切换**：大部分无明显动画，保持平滑过渡。
- **消息发送状态**：展示发送中、失败、成功，与微信一致。
- **长按操作**：列表项、消息气泡支持长按呼出 ActionSheet。
- **滑动删除 / 置顶**：列表支持左右滑动交互。

## 🔧 实现建议

1. **主题系统**
   - 在 `ThemeProvider` 内定义微信浅/深主题对象，提供色值、字体大小、圆角等变量。
   - 所有组件从主题读取外观，禁止硬编码颜色和间距。

2. **基础组件库**
   - 统一封装 `WeChatHeader`、`WeChatListItem`、`WeChatButton`、`WeChatActionSheet` 等。
   - 对话气泡、输入工具栏等采用独立组件，便于聊天页面复用。

3. **页面重构顺序**
   1. 聊天主流程（列表、聊天框、输入栏）。
   2. Tab 页与入口页（发现、我的）。
   3. 其他二级页面（设置、资料、附近的人）。
   4. 辅助页面（登录、引导、通知）。

4. **资源与图标**
   - 使用矢量图标库或自制满足微信风格的 icon set。
   - 注意图标大小（常用 24px）和线宽。

5. **多端一致性**
   - iOS/Android/Web (Expo Web) 保持视觉一致。
   - 针对不同平台的特殊控件进行等价处理。
