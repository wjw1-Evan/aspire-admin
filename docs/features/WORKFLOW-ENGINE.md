# 工作流引擎设计指南 (Workflow Engine)

> 本文档详细说明 Aspire Admin Platform 自研高性能工作流引擎的设计与实现。

## 📋 概述

工作流引擎旨在实现企业业务流程的自动化。不同于传统的线性审批，本项目支持动态分支、可视化设计、表单联动以及多维度的状态流转。

核心实现：`Platform.ApiService/Services/WorkflowEngine.cs`

## 🎯 核心概念

### 1. 流程定义 (WorkflowDefinition)
描述流程的蓝图。
- **Node**: 流程节点（开始、审批、条件、抄送、并行、结束）。
- **Edge**: 节点之间的连接线。
- **Form**: 关联的业务表单（支持 Document 或 Variable 模式）。

### 2. 流程实例 (WorkflowInstance)
流程定义的运行态。
- **Status**: 运行中、已完成、已挂起、已取消、已退回。
- **Context**: 运行时的上下文数据。

### 3. 审批记录 (WorkflowApprovalRecord)
记录每一次的人工操作。
- **Action**: 处理、驳回、转办、退回。
- **Comment**: 审批意见。

## 🛠️ 核心逻辑

### 1. 节点跳转与推进
引擎通过 `MoveToNextNodeAsync` 自动根据节点类型和条件表达式决定下一步走向：
- **审批节点**: 等待指定用户处理。支持“会签”（全通过）与“或签”（一人通过）。
- **条件节点**: 执行表达式，自动匹配下一路径。
- **并行节点**: 同时启动多个分支，等待所有分支汇合。

### 2. 表达式评估
使用 `WorkflowExpressionEvaluator` 基于业务数据（如金额、部门、申请人属性）实时计算分支条件。

### 3. 审批人解析
支持多种解析规则：
- **指定用户/角色**。
- **主管级别**: 申请人的上级主管。
- **表单字段**: 从表单数据中提取审批人。

## 🚀 关键操作

| 操作 | 描述 |
| :--- | :--- |
| **启动 (Start)** | 初始化实例，进入第一个节点。 |
| **审批 (Approve)** | 通过当前步骤，根据策略推进到下一节点。 |
| **驳回 (Reject)** | 直接终止流程，并将实例状态设为“已拒绝”。 |
| **转办 (Delegate)** | 将当前任务委派给另一名用户处理。 |
| **退回 (Return)** | 退回到流程中的指定历史节点，重新流转。 |

## 📊 监控与分析
- **WorkflowAnalyticsService**: 统计流程效率、节点耗时、待办分布。
- **WorkflowPerformanceService**: 监控引擎运行性能，识别瓶颈。

## 📝 最佳实践
- ✅ 保持流程定义版本管理。
- ✅ 复杂的业务逻辑优先使用条件节点而非人工判断。
- ✅ 为每个重要节点配置超时处理。
