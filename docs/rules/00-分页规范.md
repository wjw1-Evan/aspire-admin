# 分页处理统一规范

> 本文档是 Aspire Admin 项目的分页规范核心文档，统一定义前后端的分页类型、参数命名、接口设计等。所有后端和前端开发必须遵循本规范。

---

## 1. 统一分页类型

### 1.1 PagedResult<T> 结构

前后端统一使用以下分页响应结构：

| 属性 | 类型 | 说明 |
|------|------|------|
| `queryable` | `T[]` | 分页后的数据数组 |
| `currentPage` | `number` | 当前页码 |
| `pageSize` | `number` | 每页数量 |
| `rowCount` | `number` | 总记录数 |
| `pageCount` | `number` | 总页数 |

### 1.2 后端类型定义

```csharp
using System.Linq.Dynamic.Core;

public class PagedResult<T>
{
    public IQueryable<T> Queryable { get; set; } = Enumerable.Empty<T>().AsQueryable();
    public int CurrentPage { get; set; }
    public int PageSize { get; set; }
    public int RowCount { get; set; }
    public int PageCount { get; set; }
}
```

> 位置：`Platform.ServiceDefaults/Models/PagedResult.cs`

### 1.3 前端类型定义

```typescript
// src/types/unified-api.ts

export interface PagedResult<T> {
  queryable: T[];
  currentPage: number;
  pageSize: number;
  rowCount: number;
  pageCount: number;
}
```

## 2. 统一分页参数

### 2.1 参数命名规范

| 参数 | 说明 | 禁止使用 |
|------|------|----------|
| `page` | 当前页码（从 1 开始） | `current`、`pageIndex`、`pageNumber`、`currentPage` |
| `pageSize` | 每页数量 | `limit`、`size`、`perPage` |

### 2.2 后端请求模型

```csharp
public class PageParams
{
    public int Page { get; set; } = 1;
    public int PageSize { get; set; } = 10;
    public string? SortBy { get; set; }
    public string SortOrder { get; set; } = "desc";
    public string? Search { get; set; }
}
```

> 位置：`Platform.ServiceDefaults/Models/PageParams.cs`

### 2.3 前端请求参数

```typescript
interface PageParams {
  page?: number;       // 当前页码（从 1 开始），默认 1
  pageSize?: number;   // 每页数量，默认 10
  sortBy?: string;     // 排序字段
  sortOrder?: 'asc' | 'desc';  // 排序方向，默认 'desc'
  search?: string;     // 搜索关键词（后端自动搜索所有字符串字段）
}
```

> 位置：`Platform.Admin/src/types/page-params.ts`

## 3. 统计与分页接口分离

### 3.1 设计原则

统计数据和分页数据必须**分离为两个独立接口**：

| 接口类型 | 用途 | 数据范围 |
|----------|------|----------|
| 统计接口 | 计算全局统计 | 基于全部数据 |
| 分页接口 | 表格展示 | 只返回当前页 |

### 3.2 原因

1. 分页数据只包含当前页的记录，无法用于计算全局统计
2. 前端不应为计算统计而加载大量数据（pageSize: 1000）
3. 统计计算应在后端完成，前端直接使用

### 3.3 后端实现

```csharp
// 控制器
[HttpGet]
public async Task<IActionResult> GetRoles(
    [FromQuery] int page = 1,
    [FromQuery] int pageSize = 20)
{
    var result = await _roleService.GetRolesPaginatedAsync(page, pageSize);
    return Success(result);
}

[HttpGet("statistics")]
public async Task<IActionResult> GetRoleStatistics()
{
    var statistics = await _roleService.GetRoleStatisticsAsync();
    return Success(statistics);
}
```

### 3.4 前端实现

```typescript
// 服务层
import type { ApiResponse, PagedResult } from '@/types/unified-api';
import type { Role, RoleStatistics } from './types';

export async function getRoles(params: { page: number; pageSize: number }) {
  return request<ApiResponse<PagedResult<Role>>>('/api/roles', { params });
}

export async function getRoleStatistics() {
  return request<ApiResponse<RoleStatistics>>('/api/roles/statistics');
}

// 页面组件
const loadStatistics = async () => {
  const response = await getRoleStatistics();
  if (response.success && response.data) {
    setStatistics(response.data);
  }
};

const loadRoleData = async (params: { page: number; pageSize: number }) => {
  const response = await getRoles(params);
  if (response.success && response.data) {
    return {
      data: response.data.queryable,
      total: response.data.rowCount,
      success: true,
    };
  }
};

// 初始化时分别加载
useEffect(() => {
  loadStatistics();
}, []);
```

### 3.5 ❌ 错误做法

```typescript
// ❌ 错误：为计算统计加载大量数据
const response = await getRoles({ page: 1, pageSize: 1000 });
const data = response.data.queryable;
const stats = {
  total: data.length,
  active: data.filter(r => r.isActive).length,  // 只统计了第一页！
};
```

## 4. 后端实现规范

### 4.1 统一使用 ToPagedList 扩展方法

**所有后端分页查询必须统一使用 `ToPagedList<T>()` 扩展方法**，该方法自动整合全文搜索、动态排序和分页功能：

```csharp
using Platform.ServiceDefaults.Extensions;

public async Task<PagedResult<UserDto>> GetUsersAsync(PageParams request)
{
    var pagedResult = _context.Set<User>().ToPagedList(request);
    var items = await pagedResult.Queryable.ToListAsync();

    return new PagedResult<UserDto>
    {
        Queryable = items.Select(MapToDto).AsQueryable(),
        CurrentPage = pagedResult.CurrentPage,
        PageSize = pagedResult.PageSize,
        RowCount = pagedResult.RowCount,
        PageCount = pagedResult.PageCount
    };
}
```

### 4.2 ToPagedList 数据流

```mermaid
flowchart LR
    A["前端 Table"] -->|"pagination.current → page"| B["PageParams"]
    B -->|"POST /api/xxx/list"| C["控制器"]
    C --> D["ToPagedList"]
    D --> E["1. 全文搜索"]
    E --> F["2. 自动排序"]
    F --> G["3. 分页"]
    G --> H["PagedResult&lt;T&gt;"]
    H -->|"JSON 响应"| I["前端 PagedResult"]
```

> **前端 `pagination.current` → 后端 `page` 的转换**在各页面的 `onChange` 回调中完成，详见 [5.5 Table 参数桥接](#55-table-参数桥接)。

### 4.3 ToPagedList 特性

`ToPagedList<T>()` 方法自动完成以下功能：

1. **全文搜索**：搜索所有 `string` 类型字段（不区分大小写）。
2. **自动排序**：根据 `SortBy` 和 `SortOrder` 参数排序，如未指定则**自动按 `createdAt` 降序排序**。
3. **分页**：根据 `Page` 和 `PageSize` 参数分页。

> [!IMPORTANT]
> `ToPagedList` 会自动确保查询已排序，避免分页查询的常见错误。它是后端分页的唯一标准实现，严禁手动实现分页逻辑。

```csharp
public static System.Linq.Dynamic.Core.PagedResult<T> ToPagedList<T>(
    this IQueryable<T> query,
    Platform.ServiceDefaults.Models.PageParams? pageParams)
{
    // 1. 全文搜索
    if (!string.IsNullOrWhiteSpace(pageParams?.Search))
    {
        query = query.Where(BuildSearchFilter<T>(pageParams.Search));
    }

    // 2. 排序
    query = query.ApplySort(pageParams);

    // 3. 分页
    return query.PageResult(pageParams?.Page ?? 1, pageParams?.PageSize ?? 10);
}
```

> 位置：`Platform.ServiceDefaults/Extensions/QueryableExtensions.cs`

### 4.4 控制器层

控制器直接使用 `PageParams` 接收请求参数，**不在控制器层处理分页逻辑**：

```csharp
[HttpPost("list")]
public async Task<IActionResult> GetUsersList([FromBody] Platform.ServiceDefaults.Models.PageParams request)
{
    var result = await _userService.GetUsersWithRolesAsync(request);
    return Success(result);
}
```

> **禁止**：在控制器中手动处理分页或使用自定义查询类

## 5. 前端实现规范

### 5.1 统一使用 PageParams

前端统一使用 `@/types/page-params` 中的 `PageParams` 接口：

```typescript
import type { PageParams } from '@/types/page-params';

const searchParamsRef = useRef<PageParams>({ page: 1, pageSize: 10, search: '' });
```

### 5.2 服务层定义

```typescript
import { request } from '@umijs/max';
import type { ApiResponse, PagedResult } from '@/types/unified-api';
import type { PageParams } from '@/types/page-params';
import type { FileItem } from './types';

export async function getFileList(params: PageParams) {
  return request<ApiResponse<PagedResult<FileItem>>>('/api/files', {
    method: 'POST',
    data: params
  });
}
```

### 5.3 页面组件使用

```typescript
import type { PageParams } from '@/types/page-params';

const fetchFileList = async (params: PageParams) => {
  const response = await getFileList(params);

  if (response.success && response.data) {
    return {
      data: response.data.queryable,
      total: response.data.rowCount,
      success: true,
    };
  }

  return { data: [], total: 0, success: false };
};
```

### 5.4 访问分页数据

```typescript
const response = await getFileList({
  page: 1,
  pageSize: 20,
  search: 'keyword',
  sortBy: 'name',
  sortOrder: 'asc'
});

if (response.success && response.data) {
  const { queryable, rowCount, pageCount, currentPage, pageSize } = response.data;
  // queryable: 数据数组（后端已通过 ToPagedList 搜索和排序）
  // rowCount: 总记录数
  // pageCount: 总页数
  // currentPage: 当前页码
  // pageSize: 每页数量
}
```

### 5.5 Table 参数桥接

Ant Design 原生 `Table` 组件通过 `onChange` 回调传递分页、排序参数，需手动转换：

```typescript
// Table 的 onChange 回调
const handleTableChange = (pagination: PaginationProps, _filters: Record<string, any>, sorter: SorterResult<any>) => {
  const requestData = {
    page: pagination.current || 1,
    pageSize: pagination.pageSize || 10,
    search: searchParamsRef.current.search,
    sortBy: sorter.field as string | undefined,
    sortOrder: sorter.order === 'ascend' ? 'asc' : sorter.order === 'descend' ? 'desc' : undefined,
  };

  searchParamsRef.current = { ...searchParamsRef.current, ...requestData };
  fetchData();
};

// 响应数据处理
if (response.success && response.data) {
  const { queryable, rowCount } = response.data;
  setData(queryable || []);
  setPagination(prev => ({ ...prev, total: rowCount ?? 0 }));
}
```

> **注意**：原生 Table 使用 `pagination.current` 而非 `current`，转换应始终在各页面的 `onChange` 回调中完成。

## 6. 常见错误示例

### 6.1 后端错误

```csharp
// ❌ 错误 1：不使用 ToPagedList，手动实现分页
public async Task<PagedResult<UserDto>> GetUsersAsync(FileListQuery query)
{
    var q = _context.Set<User>().Where(x => x.Status == UserStatus.Active);

    // 手动搜索
    if (!string.IsNullOrWhiteSpace(query.Keyword))
    {
        q = q.Where(u => u.Name.Contains(query.Keyword) || u.Email.Contains(query.Keyword));
    }

    // 手动排序
    q = query.SortBy?.ToLower() switch
    {
        "name" => query.SortOrder == "desc" ? q.OrderByDescending(u => u.Name) : q.OrderBy(u => u.Name),
        _ => q.OrderByDescending(u => u.CreatedAt)
    };

    return q.PageResult(query.Page, query.PageSize);
}

// ❌ 错误 2：先 ToListAsync 再分页（内存分页）
var all = await q.ToListAsync();
return all.Skip((page - 1) * pageSize).Take(pageSize).ToList();

// ❌ 错误 3：控制器中处理分页
var users = await _service.GetAllUsersAsync();
var pagedResult = users.AsQueryable().PageResult(page, pageSize);

// ❌ 错误 4：未排序直接分页
var result = query.PageResult(page, pageSize);

// ❌ 错误 5：使用非标准参数名
public async Task<IActionResult> GetItems([FromQuery] int current, [FromQuery] int pageSize)
```

### 6.2 前端错误

```typescript
// ❌ 错误 1：不使用统一 PageParams，自定义参数
interface CustomParams {
  current: number;  // 应使用 page
  size: number;     // 应使用 pageSize
  keyword: string;  // 应使用 search
}

// ❌ 错误 2：未使用后端 ToPagedList 的搜索功能，手动过滤
const response = await getFileList({ page: 1, pageSize: 1000 });
const filtered = response.data.queryable.filter(item =>
  item.name.includes(keyword) || item.description.includes(keyword)
);

// ❌ 错误 3：未检查 success 字段
const data = response.data.queryable;

// ❌ 错误 4：重复定义已有类型
interface PagedResult<T> {
  data: T[];     // 应使用 queryable
  total: number; // 应使用 rowCount
  // 自定义的，与规范不一致
}

// ❌ 错误 5：为计算统计加载大量数据
const response = await getRoles({ page: 1, pageSize: 1000 });
const stats = { total: response.data.queryable.length };
```

## 7. 统一使用 SearchBar 组件

### 7.1 SearchBar 组件

前端统一使用 `SearchBar` 组件处理搜索表单：

```typescript
import SearchBar, { type SearchBarProps, type SearchField } from '@/components/SearchBar';
import type { PageParams } from '@/types/page-params';

const fields: SearchField[] = [
  { name: 'search', type: 'text', label: '关键词', placeholder: '输入关键词搜索' },
  { name: 'roleIds', type: 'select', label: '角色', options: [...] },
  { name: 'isActive', type: 'select', label: '状态', options: [...] },
  { name: 'dateRange', type: 'dateRange', label: '创建时间' },
];

const handleSearch = (params: PageParams) => {
  // params 已是标准格式，直接发送到后端
  request('/api/users/list', { method: 'POST', data: params });
};

<SearchBar
  fields={fields}
  onSearch={handleSearch}
  initialParams={{ pageSize: 20 }}
/>
```

### 7.2 SearchField 配置

| 类型 | 说明 | 额外处理 |
|------|------|----------|
| `text` | 文本输入 | 直接作为 `search` 参数 |
| `select` | 下拉选择 | 直接传递值 |
| `date` | 日期选择 | - |
| `dateRange` | 日期范围 | 自动转换为 `{name}Start` 和 `{name}End` |
| `number` | 数字输入 | - |
