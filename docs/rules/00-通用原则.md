# 通用原则

> 适用于 Aspire Admin 平台所有开发模块（后端、前端、移动端）。

---

## 核心架构原则

### 数据操作规范
- **全面拥抱 IDataFactory**：所有数据操作（含查询、更新、删除、跨租户操作）必须统一使用 `IDataFactory<T>`。
- **严禁直接访问数据库**：禁止注入或使用 `IMongoCollection<T>` / `IMongoDatabase`。
- **多租户自动过滤**：实现 `IMultiTenant` 的实体自动附加 `CompanyId` 过滤。

### 企业上下文唯一入口
- 除 `userId`（JWT claim）外，企业/角色/权限信息统一通过 `ITenantContext` 读取。
- 禁止从 JWT 直接取 `companyId`、`roles` 等字段。

### 菜单级权限
- 后端统一使用 `[RequireMenu("menu-name")]`（`Platform.ApiService.Attributes.RequireMenuAttribute`）做鉴权。
- 禁止旧的 `HasPermission()` / `RequirePermission()`。
- 前端不隐藏按钮实现防护，真实授权以后端判定为准。

### 统一响应格式
- HTTP API 必须返回 `ApiResponse<T>`。
- JSON 采用 camelCase 命名策略、忽略 null 字段、枚举序列化为字符串。

### 审计与软删除
- 创建/更新/删除统一走工厂的原子方法（`CreateAsync`、`FindOneAndUpdateAsync`、`FindOneAndSoftDeleteAsync`）。
- 审计字段由工厂自动维护，业务代码不得手填。

---

## 数据访问规范

### 查询构建器
- 使用 `FilterBuilder<T>`、`SortBuilder<T>`、`UpdateBuilder<T>`、`ProjectionBuilder<T>` 构建查询条件。
- 禁止手写 BsonDocument。

### 实体设计
- 业务实体应实现 `IEntity`、`ISoftDeletable`、`ITimestamped`。
- 多租户实体继承 `MultiTenantEntity`（推荐）或实现 `IMultiTenant`。
- 非多租户实体继承 `BaseEntity`。
- 集合名称优先使用 `[BsonCollectionName("xxx")]` 特性指定。
- 字段名映射优先使用 `[BsonElement("fieldName")]` 特性指定。

---

## 中间件与管道

### 中间件顺序（固定）
```
UseExceptionHandler → UseCors → UseAuthentication → UseAuthorization → UseMiddleware<ActivityLogMiddleware> → UseMiddleware<ResponseFormattingMiddleware> → MapControllers
```

### SSE 端点处理
- SSE（Server-Sent Events）端点返回 `text/event-stream`，需跳过 `ResponseFormattingMiddleware` 格式化。
- 直接写入原始 SSE 数据流。

---

## 分页规范

### 统一分页参数
- 所有分页 API 统一使用 `page`（页码，从 1 开始）和 `pageSize`（每页数量）作为查询参数。

### 参数验证
- `page` 范围 1-10000，`pageSize` 范围 1-100。
- 超出范围抛出 `ArgumentException`。

### 分页查询
- 调用 `FindPagedAsync` 时直接传递 `page` 参数，不要计算 `skip`。

### 分页响应
- 统一返回 `{ data: T[], total: number, page: number, pageSize: number }` 格式。

## 禁用模式

| 禁用项 | 正确做法 |
|--------|----------|
| 直接访问 MongoDB | 所有操作必须通过工厂 |
| 硬编码企业 ID | 必须通过 `ITenantContext` |
| 同步等待异步 | 使用 `await` |
| 缺失权限检查 | 必须使用 `[RequireMenu]` |
| 暴露底层异常 | 统一错误码与错误消息 |
| 手写 BsonDocument | 必须使用构建器 |
| 手动设置审计字段 | 由工厂自动维护 |
| `using Xxx = Yyy.Zzz` 别名 | 必须直接使用完整类型名 |
| 循环内调用单条查询 | 必须使用批量查询方法 |

## 代码洁癖

> 代码是写给人看的，顺带能在机器上运行。干净的代码减少认知负担，降低 bug 概率，提高可维护性。

### 核心理念

| 原则 | 说明 |
|------|------|
| **零警告编译** | 任何警告都是代码"污染"，必须修复 |
| **无冗余代码** | 删除不使用的变量/方法/参数 |
| **DRY 原则** | 相同逻辑不重复，提取为公共方法 |
| **命名即文档** | 变量/方法名清晰表达意图 |
| **删除而非注释** | 不用的代码直接删除，不要注释掉 |
| **文档同步** | 注释/文档与代码保持一致 |

### 具体要求

#### 编译要求
- **禁止**任何编译警告，每次提交前执行 `dotnet build` 确认 0 警告 0 错误

#### 变量使用
```csharp
// ❌ 错误：声明但未使用的变量
var userId = GetRequiredUserId();
var result = await _service.DoSomething();  // userId 未使用

// ✅ 正确：要么使用变量，要么不获取
var result = await _service.DoSomething();
```

#### 参数清理
```csharp
// ❌ 错误：XML注释有参数但方法签名没有
/// <param name="userId">用户ID</param>
public Task<Result> ExecuteAsync(Request request)

// ✅ 正确：注释与签名一致
/// <param name="request">请求参数</param>
public Task<Result> ExecuteAsync(Request request)
```

#### 审计字段
```csharp
// ❌ 错误：手动设置审计字段
entity.CreatedBy = GetRequiredUserId();
entity.UpdatedAt = DateTime.UtcNow;

// ✅ 正确：PlatformDbContext 自动维护
await _factory.CreateAsync(entity);
```

#### 表达式组合
```csharp
// ❌ 错误：可能传入 null 导致警告
filter = filter.And(p => p.Status == status);

// ✅ 正确：扩展方法支持可空参数
filter = filter.And(p => p.Status == status);
```

## Git 提交规范

### 提交信息语言
- 所有 Git 提交信息必须使用**简体中文**。
- 使用 Cursor 的 "Generate Commit Message" 功能时，必须生成中文提交信息。

### 提交信息格式
- 遵循约定式提交格式，使用中文描述。

### 提交类型
| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: 添加用户管理功能` |

## 性能优化规范

### N+1 查询防护
- **严禁**在循环内调用单条查询方法（如 `_userService.GetUserByIdAsync()`）。
- **必须**使用批量查询方法（如 `_userService.GetUsersByIdsAsync()`）。
- DTO 转换应先批量收集 ID，再批量查询，最后内存映射。

### 异常处理与日志级别
- 非关键操作的异常（如通知发送失败）应使用 `LogDebug` 记录。
- 只有需要运维关注的异常才使用 `LogWarning`。
- 日志应包含足够的上下文信息：
  ```csharp
  _logger.LogDebug(ex, "通知发送失败: TaskId={TaskId}, Action={Action}", task.Id, actionType);
  ```

## 代码质量与安全规范

### 漏洞检查
- 修改代码后必须重新检查系统漏洞并修复。
- 包括但不限于：安全漏洞（SQL 注入、XSS、CSRF、权限绕过等）、性能问题、数据泄露风险、输入验证缺陷等。

### 错误处理
- 禁止暴露底层异常信息（如堆栈、数据库错误）。
- 记录详细错误日志供运维排查。
- 返回给前端的错误消息要可读、可本地化。

---

## 实时通信规范

### SSE 实现
- 使用 Server-Sent Events（SSE）实现实时消息推送，替代 SignalR。
- 连接管理：使用 `IChatSseConnectionManager`。
- 消息广播：使用 `IChatBroadcaster` 接口。
- 心跳保活：默认间隔 30 秒。
