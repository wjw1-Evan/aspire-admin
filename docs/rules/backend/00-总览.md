# 后端总览

- 统一遵循平台级 DbContext 能力：软删除、多租户过滤、审计字段由 PlatformDbContext 实现，业务层不重复实现。
- 控制器分层：尽量避免在控制器直接使用 DbContext，将数据访问和复杂业务放到服务层封装。
- DTO 与模型分离：请求/响应 DTO 放在 Models/ 目录下，避免在控制器中定义请求 DTO。
- 权限与路由：使用 [RequireMenu] 进行菜单级鉴权，禁用早期权限判断方法（HasPermission/RequirePermission）。
- 数据访问原则：仅通过 DbContext Set<T>() 进行增删查改，避免直接操纵 Mongo 客户端。
- 提交与文档：保持命名清晰，Commit 信息以简体中文描述，遵循项目提交规范。

- 数据结构与审计字段：PlatformDbContext 保存 CreatedAt/UpdatedAt/CreatedBy/UpdatedBy、IsDeleted/DeletedAt/DeletedBy、CompanyId 等字段的自动填充。
- 多租户：通过全局查询过滤器自动应用 CompanyId，禁用手动在查询中加入 CompanyId 过滤。

- 流程相关：WorkflowController/WorkflowService 采用分层重构，逐步将复杂逻辑迁移到服务层。
