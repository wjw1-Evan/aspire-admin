# 控制器与权限

## 控制器规范

### 必须继承 BaseApiController
- **[强制]** 所有业务控制器必须继承 `BaseApiController`。
- 统一使用 `Success<T>()`、`SuccessPaged<T>()`、`ValidationError()`、`NotFoundError()` 等构造 `ApiResponse<T>`。
- **禁止**直接继承 `ControllerBase` 或返回裸 JSON。

```csharp
// ✅ 正确
public class TaskController : BaseApiController
{
    [HttpGet("{taskId}")]
    public async Task<IActionResult> GetTask(string taskId)
    {
        var task = await _taskService.GetTaskByIdAsync(taskId);
        if (task == null)
            return NotFoundError("任务", taskId);
        return Success(task);
    }
}

// ❌ 禁止
public class WrongController : ControllerBase
{
    [HttpGet]
    public async Task<User> GetUser()  // 返回裸类型
    {
        return await _userService.GetUserByIdAsync(id);
    }
}
```

### 响应方法速查

| 方法 | 使用场景 | 示例 |
|------|----------|------|
| `Success<T>(data)` | 成功返回数据 | `return Success(user);` |
| `SuccessPaged<T>(items, total, page, pageSize)` | 分页数据 | `return SuccessPaged(list, total, page, pageSize);` |
| `ValidationError(message)` | 参数验证错误 | `return ValidationError("ID 不能为空");` |
| `NotFoundError(resource, id)` | 资源不存在 | `return NotFoundError("用户", userId);` |
| `UnauthorizedError(message)` | 未授权 | `return UnauthorizedError("登录已过期");` |
| `ForbiddenError(message)` | 禁止访问 | `return ForbiddenError("无权访问此资源");` |
| `ServerError(message)` | 服务器错误 | `return ServerError("系统繁忙");` |

## 权限控制规范

### 必须使用 RequireMenu
- **[强制]** 敏感操作必须添加 `[RequireMenu("module:resource")]`。
- 权限标识使用 `:` 分隔，格式为 `模块:资源:操作`。

```csharp
// ✅ 正确：使用 RequireMenu
[HttpPost("create")]
[RequireMenu("task-management:create")]
public async Task<IActionResult> CreateTask([FromBody] CreateTaskRequest request) { }

[HttpPut("{id}/activate")]
[RequireMenu("user-management")]  // 用户管理类操作
public async Task<IActionResult> ActivateUser(string id) { }

// ❌ 禁止：使用废弃方法
[RequirePermission("users.create")]
public async Task<IActionResult> CreateUser([FromBody] CreateUserRequest request) { }
```

### 用户上下文获取

#### 获取当前用户 ID
使用 `BaseApiController` 提供的属性和方法获取当前用户信息：

```csharp
public class PasswordBookController : BaseApiController
{
    // ✅ 正确：使用基类提供的方法
    protected string? CurrentUserId => GetCurrentUserId();        // 可空
    protected string GetRequiredUserId();                          // 抛异常
    
    // ✅ 正确：获取企业 ID
    protected Task<string?> GetCurrentCompanyIdAsync();
    protected Task<string> GetRequiredCompanyIdAsync();
    
    // ✅ 正确：检查登录状态
    protected bool IsAuthenticated => !string.IsNullOrEmpty(CurrentUserId);
}
```

#### 获取企业上下文
通过 `ITenantContext` 获取企业信息：

```csharp
// ✅ 正确：通过 ITenantContext 获取
var companyId = await GetCurrentCompanyIdAsync();
var isAdmin = await _tenantContext.IsAdminAsync();

// ❌ 禁止：直读 JWT Claims
var companyId = User.FindFirst("companyId")?.Value;
```

#### 用户 ID 传递给服务
控制器获取用户 ID 后，应传递给服务层处理：

```csharp
// ✅ 正确：控制器获取 userId 并传递给服务
public async Task<IActionResult> CreateEntry([FromBody] CreateRequest request)
{
    var userId = GetRequiredUserId();  // 控制器获取
    var result = await _service.CreateEntryAsync(request, userId);  // 传递给服务
    return Success(result);
}

// ❌ 禁止：在服务中直接使用 ITenantContext 获取 userId
public async Task CreateEntryAsync(CreateRequest request)
{
    var userId = _tenantContext.GetCurrentUserId();  // 不推荐
}
```

## 菜单定义规范

### 菜单名称（Menu.Name 字段）
- 统一使用连字符 `-` 分隔，格式为 `模块-资源`。
- 示例：`workflow-list`、`workflow-monitor`、`project-management-task`

### 权限标识（RequireMenu 参数和 Menu.Permissions 字段）
- 使用冒号 `:` 分隔，格式为 `模块:资源:操作`。
- 示例：`workflow:list`、`workflow:create`、`document:list`

### 菜单定义位置
- 所有系统菜单在 `Platform.DataInitializer/Services/DataInitializerService.cs` 的 `GetExpectedMenus` 方法中定义。

### 子菜单映射
- 新增子菜单时，必须在 `GetParentMenuNameByChildName` 方法中添加映射关系。

### 前端翻译
- 菜单翻译键格式为 `menu.模块.资源`（使用 `.` 分隔）。
- 在 `src/locales/*/menu.ts` 中维护。

## 常见错误示例

```csharp
// ❌ 错误 1：缺少 RequireMenu
[HttpPut("{id}/deactivate")]
public async Task<IActionResult> DeactivateUser(string id)
{
    await _userService.DeactivateUserAsync(id);
    return Success("用户已禁用");
}

// ✅ 正确：添加 RequireMenu
[HttpPut("{id}/deactivate")]
[RequireMenu("user-management")]
public async Task<IActionResult> DeactivateUser(string id)
{
    await _userService.DeactivateUserAsync(id);
    return Success("用户已禁用");
}

// ❌ 错误 2：直读 JWT
[HttpGet("list")]
public async Task<IActionResult> GetUsers()
{
    var companyId = User.Claims.FirstOrDefault(c => c.Type == "companyId")?.Value;
    var users = await _userService.GetUsersAsync(companyId);
    return Success(users);
}

// ✅ 正确：通过 ITenantContext
[HttpGet("list")]
[RequireMenu("user-management:list")]
public async Task<IActionResult> GetUsers()
{
    var companyId = await _tenantContext.GetCurrentCompanyIdAsync();
    var users = await _userService.GetUsersAsync(companyId);
    return Success(users);
}
```
