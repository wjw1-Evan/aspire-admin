# 数据访问与审计

## 数据访问核心原则

### 严禁修改 PlatformDbContext
- **[强制]** 严禁修改 `Platform.ServiceDefaults/Services/PlatformDbContext.cs` 文件。
- 所有数据库配置、全局查询过滤器和拦截逻辑在此文件中统一管理。
- 业务扩展应通过实体特性或接口完成。

### 必须注入 DbContext
- 所有数据的新增、查询、修改、删除**必须**通过注入 `DbContext` 来完成。我们在 `ServiceExtensions.cs` 中已经将底层的 `PlatformDbContext` 注册为 `DbContext`，使用基类注入即可自动享受底层的高级功能。
- **禁止**直接注入 `IMongoCollection<T>` 或 `IMongoDatabase`，以防绕过多租户过滤和软删除拦截。

```csharp
// ✅ 正确：通过构造函数注入 DbContext，并使用 Set<T>() 操作
using Microsoft.EntityFrameworkCore;

public class TaskService : ITaskService
{
    private readonly DbContext _context;

    public TaskService(DbContext context)
    {
        _context = context;
    }
}

// ❌ 禁止：直接注入 IMongoCollection
public class WrongService
{
    private readonly IMongoCollection<WorkTask> _collection;

    public WrongService(IMongoCollection<WorkTask> collection)
    {
        _collection = collection;
    }
}
```

## 核心操作方法 (DbContext)

### 创建操作
```csharp
// 创建单个实体
var user = new User { Username = "test" };
await _context.Set<User>().AddAsync(user);
await _context.SaveChangesAsync();

// 批量创建
await _context.Set<User>().AddRangeAsync(user1, user2, user3);
await _context.SaveChangesAsync();
```

### 更新操作
```csharp
// 查找并更新（自动设置 UpdatedAt、UpdatedBy）
var user = await _context.Set<User>().FirstOrDefaultAsync(u => u.Id == userId);
if (user != null)
{
    user.Username = "new_username";
    await _context.SaveChangesAsync();
}
```

### 删除操作（软删除）

**重要**：`DbContext.Remove()` 方法会被 PlatformDbContext 统一转换为软删除，无需手动设置 `IsDeleted = true`。

```csharp
// ✅ 正确：直接使用 Remove，由 PlatformDbContext 自动转换为软删除
var user = await _context.Set<User>().FirstOrDefaultAsync(u => u.Id == userId);
if (user != null)
{
    _context.Set<User>().Remove(user);  // 自动设置 IsDeleted=true, DeletedAt, DeletedBy
    await _context.SaveChangesAsync();
}

// ❌ 不推荐：手动设置 IsDeleted（虽然也能工作，但不必要的冗余）
var user = await _context.Set<User>().FirstOrDefaultAsync(u => u.Id == userId);
if (user != null)
{
    user.IsDeleted = true; // 触发底层拦截更改标记
    await _context.SaveChangesAsync();
}

// 批量软删除
var usersToDelete = await _context.Set<User>()
    .Where(u => u.Status == "Inactive")
    .ToListAsync();
foreach(var u in usersToDelete)
{
    _context.Set<User>().Remove(u);  // 自动转换为软删除
}
await _context.SaveChangesAsync();
```

## 查询方法

### 基础查询
```csharp
// 基础查询数据（多租户过滤与软删除由 PlatformDbContext 全局查询过滤器自动处理）
var items = await _context.Set<WorkTask>().ToListAsync();

// 带过滤条件
var pendingItems = await _context.Set<WorkTask>()
    .Where(e => e.Status == TaskStatus.Pending)
    .ToListAsync();

// 带排序和限制
var topItems = await _context.Set<WorkTask>()
    .Where(e => e.Status == TaskStatus.Pending)
    .OrderByDescending(e => e.CreatedAt)
    .Take(10)
    .ToListAsync();

// 分页查询
var page = 1;
var pageSize = 10;
var query = _context.Set<WorkTask>().Where(e => e.Status == TaskStatus.Pending);

var total = await query.LongCountAsync();
var pagedItems = await query
    .OrderByDescending(e => e.CreatedAt)
    .Skip((page - 1) * pageSize)
    .Take(pageSize)
    .ToListAsync();
```

### 用户数据过滤
用户数据过滤在**服务层**通过 LINQ 条件实现：
```csharp
// 查询当前用户创建的数据
var myItems = await _context.Set<WorkTask>()
    .Where(e => e.CreatedBy == currentUserId)
    .ToListAsync();

// 查询自己的 + 公有数据（如密码本）
var items = await _context.Set<PasswordBookEntry>()
    .Where(e => e.UserId == currentUserId || e.IsPublic)
    .ToListAsync();
```

## 审计字段自动维护

### PlatformDbContext 自动维护原理

`PlatformDbContext.SaveChangesAsync()` 在保存时会自动维护以下审计字段：

| 字段 | 说明 | 设置时机 |
|------|------|----------|
| `CreatedAt` | 创建时间 | 新增实体时 |
| `UpdatedAt` | 更新时间 | 新增/更新实体时 |
| `CreatedBy` | 创建人 | 新增实体时，从 `ITenantContext` 获取 |
| `UpdatedBy` | 更新人 | 新增/更新实体时，从 `ITenantContext` 获取 |
| `IsDeleted` | 软删除标记 | 软删除时 (当代码里显式赋 true 时会补全信息) |
| `DeletedAt` | 删除时间 | 软删除时 |
| `DeletedBy` | 删除人 | 软删除时 |
| `CompanyId` | 企业ID | 新增 `IMultiTenant` 实体时 |

### 禁止手动赋值

**禁止**在控制器或服务中手动设置审计字段，包括但不限于：

```csharp
// ❌ 禁止：手动设置
entity.CreatedBy = _tenantContext.GetCurrentUserId();
entity.CreatedAt = DateTime.UtcNow;
entity.UpdatedAt = DateTime.UtcNow;

// ✅ 正确：PlatformDbContext 自动维护
await _context.Set<Entity>().AddAsync(entity); // 所有审计字段自动设置
await _context.SaveChangesAsync();
```

### 特殊字段说明
- **`LastOperationAt`**、`**LastOperationType**`：由 `IOperationTrackable` 接口自动维护，记录最后操作时间和类型。
- **`JoinedAt`**、**`ApprovedAt`** 等：业务时间字段需要手动设置。

### 服务获取用户 ID 方式
**重要**：如果不涉及专属“我创建的”或业务规则，服务中不需要获取 userId 或 companyId，应完全依赖 PlatformDbContext 全局过滤。

#### 何时可在服务中使用 ITenantContext
**一般禁止**在服务中主动拼装过滤逻辑，由于多租户全自动处理。
仅在以下极特殊场景可以使用 `ITenantContext`：
1. **业务逻辑需要当前用户信息** - 如 "我的文档"、"待我审批" 等专属筛选
2. **后台服务** - 无控制器上下文的强制获取补充（比如批量创建任务或背景任务补充企业ID上下游）
3. **MCP 工具** - AI 代理需要获取企业上下文作为指令

### 服务中禁止授权检查
**重要**：服务层不应进行创建者/负责人基础授权检查，应依赖 Controller 里的菜单级权限 `[RequireMenu]`。

## 多租户隔离

### 自动隔离原理
`PlatformDbContext` 的 `OnModelCreating` 配置了全局查询过滤器：
- 自动为所有 `IMultiTenant` 实体在查询时附加 `CompanyId` 匹配以及 `IsDeleted = false`。
- 在保存时 (`SaveChanges`) 为新增的 `IMultiTenant` 自动回填当前 `CompanyId`。

### 业务层不需要考虑隔离
```csharp
// ✅ 正确：不需要手动处理租户隔离，底层已拦截
var projects = await _context.Set<Project>().ToListAsync();

// ✅ 正确：不需要手动写入租户ID
var project = new Project { Name = "新项目" };
await _context.Set<Project>().AddAsync(project);
await _context.SaveChangesAsync();
```

### 跨租户查询（慎用）
仅在极少数运维/后台计划任务下才允许使用 `IgnoreQueryFilters()`，通常用于系统层级获取数据：
```csharp
// ⚠️ 慎用：跨租户查询（无视CompanyId及IsDeleted）
var user = await _context.Set<User>().IgnoreQueryFilters().FirstOrDefaultAsync(u => u.Id == userId);
```

## 实体基类与接口

### BaseEntity
适用于非多租户实体（如 `Company`、`AppUser`、`RefreshToken`）：
```csharp
public class AppUser : BaseEntity
{
    public string Username { get; set; }
    public string Email { get; set; }
}
```

### MultiTenantEntity
适用于多租户实体（如 `Role`、`Project`、`WorkTask`）：
```csharp
public class WorkTask : MultiTenantEntity
{
    public string TaskName { get; set; }
    public TaskStatus Status { get; set; }
}
```

### 接口要求
- 所有 MongoDB 实体必须实现 `IEntity`、`ISoftDeletable`、`ITimestamped`。
- 多租户实体需实现 `IMultiTenant` 或继承 `MultiTenantEntity`。

### 集合命名与字段名映射
- 优先使用 `[BsonCollectionName("xxx")]` 特性指定集合名。
- 优先使用 `[BsonElement("fieldName")]` 特性指定 BSON 字段名。
