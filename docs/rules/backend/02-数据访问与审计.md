# 数据访问与审计

## 数据访问核心原则

### 严禁修改 PlatformDbContext
- **[强制]** 严禁修改 `Platform.ServiceDefaults/Services/PlatformDbContext.cs` 文件。
- 所有数据库配置、拦截器和过滤逻辑在此文件中统一管理。
- 业务扩展应通过实体特性、接口或 `IDataFactory` 完成。

### 必须使用 IDataFactory
- 所有数据操作必须使用 `IDataFactory<T>`。
- **禁止**直接注入 `IMongoCollection<T>`、`IMongoDatabase` 或 `DbContext`。

```csharp
// ✅ 正确：通过构造函数注入 IDataFactory
public class TaskService : ITaskService
{
    private readonly IDataFactory<WorkTask> _taskFactory;
    private readonly IDataFactory<TaskExecutionLog> _executionLogFactory;

    public TaskService(
        IDataFactory<WorkTask> taskFactory,
        IDataFactory<TaskExecutionLog> executionLogFactory)
    {
        _taskFactory = taskFactory;
        _executionLogFactory = executionLogFactory;
    }
}

// ❌ 禁止：直接注入 IMongoCollection
public class WrongService
{
    private readonly IMongoCollection<WorkTask> _collection;

    public WrongService(IMongoCollection<WorkTask> collection)
    {
        _collection = collection;
    }
}
```

## 原子操作方法

### 创建操作
```csharp
// 创建单个实体
var user = await _userFactory.CreateAsync(new User { Username = "test" });

// 批量创建
var users = await _userFactory.CreateManyAsync(new[] { user1, user2, user3 });
```

### 更新操作
```csharp
// 查找并更新（自动设置 UpdatedAt、UpdatedBy）
var updated = await _userFactory.UpdateAsync(userId, u =>
{
    u.Username = "new_username";
    u.UpdatedAt = DateTime.UtcNow;
});

// 查找并替换（原子替换整个文档）
var replaced = await _userFactory.ReplaceAsync(userId, newUser);
```

### 删除操作
```csharp
// 软删除（自动设置 IsDeleted、DeletedAt、DeletedBy）
await _userFactory.SoftDeleteAsync(userId);

// 批量软删除
await _userFactory.SoftDeleteManyAsync(filter, deletedBy);
```

## 审计字段自动维护

### 工厂自动维护的字段
- `CreatedAt`、`UpdatedAt`：时间戳
- `IsDeleted`、`DeletedAt`、`DeletedBy`：软删除
- `CreatedBy`、`UpdatedBy`：审计人

### 禁止手动赋值
- **禁止**在业务代码中手工设置上述审计字段。

```csharp
// ❌ 禁止：手动设置审计字段
var user = new User
{
    Username = "test",
    CreatedAt = DateTime.UtcNow,  // 禁止
    CreatedBy = currentUserId      // 禁止
};

// ✅ 正确：工厂自动维护
var user = new User { Username = "test" };
await _userFactory.CreateAsync(user);  // 审计字段自动设置
```

### 统一使用用户 ID
- 废弃 `CreatedByUsername` 和 `UpdatedByUsername`，统一使用用户 ID。
- 通过 `ITenantContext` 获取当前用户 ID。

## 跨租户访问

### 默认行为
- `IDataFactory<T>` 会为实现 `IMultiTenant` 的实体自动附加 `CompanyId` 过滤。
- 始终附加 `IsDeleted = false` 软删除过滤。

### 跨租户查询（慎用）
- 仅在极少数运维/平台级场景下才允许使用 `WithoutTenantFilter` 方法。
- 必须拥有明确的运维/平台级菜单权限。
- 必须在上层服务中记录详细审计日志。

```csharp
// ✅ 正确：普通查询（自动租户过滤）
var users = await _userFactory.FindAsync(u => u.CompanyId == companyId);

// ⚠️ 慎用：跨租户查询
var user = await _userFactory.GetByIdWithoutTenantFilterAsync(userId);
```

## 多企业数据隔离统一获取方式

### 统一获取 CompanyId

所有控制器必须通过 `BaseApiController` 提供的统一方法获取当前企业 ID，禁止直接从 JWT Claim 或用户对象中读取。

```csharp
// ✅ 正确：使用 BaseApiController 提供的统一方法
public class MyController : BaseApiController
{
    private readonly ITenantContext _tenantContext;

    public MyController(ITenantContext tenantContext)
    {
        _tenantContext = tenantContext;
    }

    [HttpGet]
    public async Task<IActionResult> GetData()
    {
        // 方法1：获取可选的 CompanyId
        var companyId = await GetCurrentCompanyIdAsync();

        // 方法2：获取必需的 CompanyId（未找到时抛异常）
        var requiredCompanyId = await GetRequiredCompanyIdAsync();

        // 方法3：直接使用 ITenantContext
        var ctxCompanyId = await _tenantContext.GetCurrentCompanyIdAsync();

        return Success(data);
    }
}
```

### 禁止的写法

```csharp
// ❌ 禁止：从 JWT Claim 直接读取 companyId
var companyId = User.FindFirst("companyId")?.Value;

// ❌ 禁止：直接从用户对象读取 CompanyId
var user = await _userService.GetUserByIdAsync(userId);
var companyId = user.CurrentCompanyId;  // 不推荐
```

### 何时可以使用用户对象

仅在以下情况可以从数据库查询用户对象：
1. 需要获取用户的其他业务信息（如角色、权限等）
2. 需要实现复杂的回退策略（如 CompanyController 中的企业切换逻辑）

但仍然建议在获取 CompanyId 时使用统一方法：

```csharp
// ✅ 推荐：先获取 CompanyId，再查询其他用户信息
var companyId = await GetRequiredCompanyIdAsync();
var user = await _userService.GetUserByIdAsync(userId);
var roles = user.Roles; // 使用用户其他信息
```

### 特殊场景

**CompanyController.GetCurrentCompany** 是一个特殊端点，需要处理企业切换、回退策略等复杂逻辑，因此可以使用直接查询用户的方式：

```csharp
// 此方法为特殊处理，不适用于普通业务控制器
var user = await userFactory.GetByIdAsync(userId);
var companyId = user.CurrentCompanyId;
// 回退策略...
```

## 实体基类与接口

### BaseEntity
适用于非多租户实体（如 `Company`、`AppUser`、`RefreshToken`）：
```csharp
public class AppUser : BaseEntity
{
    public string Username { get; set; }
    public string Email { get; set; }
}
```

### MultiTenantEntity
适用于多租户实体（如 `Role`、`Project`、`WorkTask`）：
```csharp
public class WorkTask : MultiTenantEntity
{
    public string TaskName { get; set; }
    public TaskStatus Status { get; set; }
}
```

### 接口要求
- 所有 MongoDB 实体必须实现 `IEntity`、`ISoftDeletable`、`ITimestamped`。
- 多租户实体需实现 `IMultiTenant` 或继承 `MultiTenantEntity`。

### 集合命名
- 优先使用 `[BsonCollectionName("xxx")]` 特性指定集合名。
- 否则使用类型名的复数形式（小写）。

```csharp
[BsonCollectionName("work_tasks")]
public class WorkTask : MultiTenantEntity { }
```

### 字段名映射
- 优先使用 `[BsonElement("fieldName")]` 特性指定 BSON 字段名。
- 构建器会自动识别并使用正确的字段名。

```csharp
public class WorkTask : MultiTenantEntity
{
    [BsonElement("task_name")]
    public string TaskName { get; set; }
}
```
