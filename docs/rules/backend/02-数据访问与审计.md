# 数据访问与审计

## 数据访问核心原则

### 严禁修改 PlatformDbContext
- **[强制]** 严禁修改 `Platform.ServiceDefaults/Services/PlatformDbContext.cs` 文件。
- 所有数据库配置、拦截器和过滤逻辑在此文件中统一管理。
- 业务扩展应通过实体特性、接口或 `IDataFactory` 完成。

### 必须使用 IDataFactory
- 所有数据操作必须使用 `IDataFactory<T>`。
- **禁止**直接注入 `IMongoCollection<T>`、`IMongoDatabase` 或 `DbContext`。

```csharp
// ✅ 正确：通过构造函数注入 IDataFactory
public class TaskService : ITaskService
{
    private readonly IDataFactory<WorkTask> _taskFactory;
    private readonly IDataFactory<TaskExecutionLog> _executionLogFactory;

    public TaskService(
        IDataFactory<WorkTask> taskFactory,
        IDataFactory<TaskExecutionLog> executionLogFactory)
    {
        _taskFactory = taskFactory;
        _executionLogFactory = executionLogFactory;
    }
}

// ❌ 禁止：直接注入 IMongoCollection
public class WrongService
{
    private readonly IMongoCollection<WorkTask> _collection;

    public WrongService(IMongoCollection<WorkTask> collection)
    {
        _collection = collection;
    }
}
```

## 原子操作方法

### 创建操作
```csharp
// 创建单个实体
var user = await _userFactory.CreateAsync(new User { Username = "test" });

// 批量创建
var users = await _userFactory.CreateManyAsync(new[] { user1, user2, user3 });
```

### 更新操作
```csharp
// 查找并更新（自动设置 UpdatedAt、UpdatedBy）
var updated = await _userFactory.UpdateAsync(userId, u =>
{
    u.Username = "new_username";
    u.UpdatedAt = DateTime.UtcNow;
});

// 查找并替换（原子替换整个文档）
var replaced = await _userFactory.ReplaceAsync(userId, newUser);
```

### 删除操作
```csharp
// 软删除（自动设置 IsDeleted、DeletedAt、DeletedBy）
await _userFactory.SoftDeleteAsync(userId);

// 批量软删除
await _userFactory.SoftDeleteManyAsync(filter, deletedBy);
```

## 查询方法

### 基础查询
```csharp
// 查询数据（多租户过滤由 PlatformDbContext 全局查询过滤器自动处理）
var items = await _factory.FindAsync();

// 带过滤条件
var items = await _factory.FindAsync(e => e.Status == TaskStatus.Pending);

// 带排序和限制
var items = await _factory.FindAsync(
    filter: e => e.Status == TaskStatus.Pending,
    orderBy: q => q.OrderByDescending(e => e.CreatedAt),
    limit: 10);

// 分页查询
var (items, total) = await _factory.FindPagedAsync(
    filter: e => e.Status == TaskStatus.Pending,
    orderBy: q => q.OrderByDescending(e => e.CreatedAt),
    page: 1,
    pageSize: 10);
```

### 用户数据过滤
用户数据过滤在**服务层**通过 filter 参数实现：
```csharp
// 查询当前用户创建的数据
var myItems = await _factory.FindAsync(
    filter: e => e.CreatedBy == currentUserId);

// 查询自己的 + 公有数据（如密码本）
var items = await _factory.FindAsync(
    filter: e => e.UserId == currentUserId || e.IsPublic);
```

### 多租户过滤原理
`PlatformDbContext` 的 `OnModelCreating` 配置了全局查询过滤器：
- 为所有 `IMultiTenant` 实体自动附加 `CompanyId` 过滤
- 为所有实体附加 `IsDeleted = false` 软删除过滤

数据工厂只负责执行 filter 表达式，不过滤逻辑由调用方在服务层控制。

## 审计字段自动维护

### PlatformDbContext 自动维护原理

`PlatformDbContext.SaveChangesAsync()` 在保存时会自动维护以下审计字段：

| 字段 | 说明 | 设置时机 |
|------|------|----------|
| `CreatedAt` | 创建时间 | 新增实体时 |
| `UpdatedAt` | 更新时间 | 新增/更新实体时 |
| `CreatedBy` | 创建人 | 新增实体时，从 `ITenantContext` 获取 |
| `UpdatedBy` | 更新人 | 新增/更新实体时，从 `ITenantContext` 获取 |
| `IsDeleted` | 软删除标记 | 软删除时 |
| `DeletedAt` | 删除时间 | 软删除时 |
| `DeletedBy` | 删除人 | 软删除时 |
| `CompanyId` | 企业ID | 新增 `IMultiTenant` 实体时 |

### 禁止手动赋值

**禁止**在控制器或服务中手动设置审计字段，包括但不限于：

```csharp
// ❌ 禁止：控制器中手动设置
knowledgeBase.CreatedBy = GetRequiredUserId();
kb.UpdatedBy = GetRequiredUserId();
kb.UpdatedAt = DateTime.UtcNow;

// ❌ 禁止：服务中手动设置
entity.CreatedBy = _tenantContext.GetCurrentUserId();
entity.CreatedAt = DateTime.UtcNow;
entity.UpdatedAt = DateTime.UtcNow;

// ✅ 正确：PlatformDbContext 自动维护
await _factory.CreateAsync(entity);  // 所有审计字段自动设置
await _factory.UpdateAsync(id, e => { e.Name = "新名称"; });  // UpdatedAt/UpdatedBy 自动设置
await _factory.SoftDeleteAsync(id);  // DeletedAt/DeletedBy/IsDeleted 自动设置
```

### 创建实体时

```csharp
// ✅ 正确：无需设置任何审计字段
var user = new User
{
    Username = "test",
    Email = "test@example.com"
};
await _userFactory.CreateAsync(user);  // CreatedAt/UpdatedAt/CreatedBy/UpdatedBy 自动设置
```

### 更新实体时

```csharp
// ✅ 正确：无需手动设置审计字段
await _userFactory.UpdateAsync(userId, u =>
{
    u.Email = "new@example.com";
    // UpdatedAt/UpdatedBy 自动设置
});
```

### 特殊字段说明

- **`LastOperationAt`**、`**LastOperationType**`：由 `IOperationTrackable` 接口自动维护，记录最后操作时间和类型
- **`JoinedAt`**：业务字段（如用户加入企业时间），需要手动设置
- **`ApprovedAt`**：业务字段（如审批时间），需要手动设置

### 服务获取用户 ID 方式

**重要**：服务中不需要获取 userId，应完全依赖 PlatformDbContext 全局过滤。

```csharp
// ✅ 正确：服务不接收 userId/companyId 参数
public class DocumentService : IDocumentService
{
    private readonly IDataFactory<Document> _documentFactory;

    public async Task<Document> CreateDocumentAsync(CreateDocumentRequest request)
    {
        // PlatformDbContext 自动维护审计字段
        var document = new Document { Title = request.Title };
        return await _documentFactory.CreateAsync(document);
    }

    public async Task<(List<Document> items, long total)> GetDocumentsAsync(DocumentQueryParams query)
    {
        // 自动按企业过滤，无需手动添加 CompanyId 条件
        return await _documentFactory.FindPagedAsync(
            filter: d => true,  // PlatformDbContext 自动过滤
            orderBy: q => q.OrderByDescending(d => d.CreatedAt),
            query.Page,
            query.PageSize);
    }
}
```

#### 何时可在服务中使用 ITenantContext

**禁止**在服务中注入和使用 `ITenantContext`，完全依赖 PlatformDbContext 全局过滤。

仅在以下极特殊场景可以使用：
1. **业务逻辑需要当前用户信息** - 如 "我的文档"、"待我审批" 等筛选
2. **后台服务** - 无控制器上下文的场景
3. **MCP 工具** - AI 代理需要获取企业上下文

```csharp
// ✅ 可以：业务逻辑需要当前用户（如"我的"筛选）
public async Task<(List<Document> items, long total)> GetDocumentsAsync(DocumentQueryParams query)
{
    var userId = _tenantContext.GetCurrentUserId();
    
    if (query.FilterType == "my")
    {
        return await _documentFactory.FindPagedAsync(
            filter: d => d.CreatedBy == userId);
    }
    return await _documentFactory.FindPagedAsync(filter: d => true);
}
```

### 统一使用用户 ID
- 废弃 `CreatedByUsername` 和 `UpdatedByUsername`，统一使用用户 ID。
- 通过 `ITenantContext` 获取当前用户 ID（仅限 PlatformDbContext 内部）。

### 服务中禁止授权检查

**重要**：服务层不应进行创建者/负责人授权检查，应依赖菜单级权限 `[RequireMenu]`。

```csharp
// ❌ 禁止：服务中进行创建者授权检查
public async Task<Document?> UpdateDocumentAsync(string id, UpdateDocumentRequest request)
{
    var currentUserId = _tenantContext.GetCurrentUserId();
    var document = await _documentFactory.GetByIdAsync(id);
    
    // 禁止：创建者授权检查
    if (document.CreatedBy != currentUserId)
        throw new UnauthorizedAccessException("无权修改他人的文档");
}

// ✅ 正确：服务不进行授权检查，依赖 [RequireMenu] 权限
public async Task<Document?> UpdateDocumentAsync(string id, UpdateDocumentRequest request)
{
    var document = await _documentFactory.GetByIdAsync(id);
    if (document == null) return null;
    
    await _documentFactory.UpdateAsync(id, d => d.Title = request.Title);
    return await _documentFactory.GetByIdAsync(id);
}
```

**控制器层**：
```csharp
// ✅ 正确：通过 [RequireMenu] 进行菜单级授权
[HttpPut("{id}")]
[RequireMenu("document-edit")]  // 菜单级权限控制
public async Task<IActionResult> UpdateDocument(string id, [FromBody] UpdateDocumentRequest request)
{
    var document = await _documentService.UpdateDocumentAsync(id, request);
    return Success(document);
}
```

## 多租户隔离

### 自动隔离原理

`PlatformDbContext` 的 `OnModelCreating` 配置了全局查询过滤器：

```csharp
// 自动为所有 IMultiTenant 实体附加 CompanyId 过滤
if (typeof(IMultiTenant).IsAssignableFrom(type))
{
    var companyIdProperty = Expression.Property(parameter, nameof(IMultiTenant.CompanyId));
    var currentCompanyIdProperty = Expression.Property(Expression.Constant(this), nameof(CurrentCompanyId));
    var tenantFilter = Expression.Equal(companyIdProperty, currentCompanyIdProperty);
    filterBody = filterBody == null ? tenantFilter : Expression.AndAlso(filterBody, tenantFilter);
}
```

**效果**：
- 查询 `IMultiTenant` 实体时，自动附加 `CompanyId` 过滤
- 新增实体时，自动设置 `CompanyId`
- 所有查询自动附加 `IsDeleted = false` 软删除过滤

### 业务层不需要考虑隔离

```csharp
// ✅ 正确：不需要手动处理租户隔离
var projects = await _projectFactory.FindAsync();  // 自动按企业过滤

var project = await _projectFactory.CreateAsync(new Project
{
    Name = "新项目"  // CompanyId 自动设置
});

// ✅ 正确：不需要在 filter 中添加 CompanyId
var (items, total) = await _factory.FindPagedAsync(
    filter: d => d.Status == DocumentStatus.Draft,  // 无需 d.CompanyId == companyId
    orderBy: q => q.OrderByDescending(d => d.CreatedAt),
    page, pageSize);
```

### 何时可以使用 ITenantContext

**禁止**在服务中注入 `ITenantContext`，完全依赖 PlatformDbContext 全局过滤。

**禁止**：
- 在查询 filter 中手动添加 CompanyId 条件（已自动处理）
- 传递 companyId 参数给数据工厂方法
- 在服务中进行创建者/负责人授权检查

**极特殊场景**可使用（需注释说明原因）：
1. **业务逻辑需要当前用户** - 如 "我的文档"、"待我审批" 筛选
2. **后台服务** - 无控制器上下文的场景
3. **MCP 工具** - AI 代理需要获取企业上下文
4. **PlatformDbContext 内部** - 自动维护审计字段

### 数据工厂自动处理原理

`PlatformDbContext` 的 `OnModelCreating` 方法中配置了全局查询过滤器：

```csharp
// 自动为所有 IMultiTenant 实体附加 CompanyId 过滤
if (typeof(IMultiTenant).IsAssignableFrom(type))
{
    var companyIdProperty = Expression.Property(parameter, nameof(IMultiTenant.CompanyId));
    var currentCompanyIdProperty = Expression.Property(Expression.Constant(this), nameof(CurrentCompanyId));
    var tenantFilter = Expression.Equal(companyIdProperty, currentCompanyIdProperty);
    filterBody = filterBody == null ? tenantFilter : Expression.AndAlso(filterBody, tenantFilter);
}
```

同时，`SaveChangesAsync` 时会自动为新增实体的 `CompanyId` 赋值：

```csharp
// 新增实体时自动设置 CompanyId
if (isAdded && entity is IMultiTenant tenant && string.IsNullOrEmpty(tenant.CompanyId))
{
    tenant.CompanyId = currentCompanyId ?? string.Empty;
}
```

### 特殊场景

仅在以下情况可以手动处理 CompanyId：
1. **CompanyController** - 企业切换逻辑需要获取用户当前企业
2. **系统监控** - 获取全局数据库统计信息
3. **MCP 工具** - AI 代理需要获取企业上下文

> **注意**：服务中应避免使用 ITenantContext，完全依赖 PlatformDbContext 全局过滤。

### 跨租户查询（慎用）

仅在极少数运维/平台级场景下才允许使用 `WithoutTenantFilter` 方法：

```csharp
// ⚠️ 慎用：跨租户查询
var user = await _userFactory.GetByIdWithoutTenantFilterAsync(userId);
```

## 实体基类与接口

### BaseEntity
适用于非多租户实体（如 `Company`、`AppUser`、`RefreshToken`）：
```csharp
public class AppUser : BaseEntity
{
    public string Username { get; set; }
    public string Email { get; set; }
}
```

### MultiTenantEntity
适用于多租户实体（如 `Role`、`Project`、`WorkTask`）：
```csharp
public class WorkTask : MultiTenantEntity
{
    public string TaskName { get; set; }
    public TaskStatus Status { get; set; }
}
```

### 接口要求
- 所有 MongoDB 实体必须实现 `IEntity`、`ISoftDeletable`、`ITimestamped`。
- 多租户实体需实现 `IMultiTenant` 或继承 `MultiTenantEntity`。

### 集合命名
- 优先使用 `[BsonCollectionName("xxx")]` 特性指定集合名。
- 否则使用类型名的复数形式（小写）。

```csharp
[BsonCollectionName("work_tasks")]
public class WorkTask : MultiTenantEntity { }
```

### 字段名映射
- 优先使用 `[BsonElement("fieldName")]` 特性指定 BSON 字段名。
- 构建器会自动识别并使用正确的字段名。

```csharp
public class WorkTask : MultiTenantEntity
{
    [BsonElement("task_name")]
    public string TaskName { get; set; }
}
```

### 自定义用户字段

适用于需要自定义用户拥有者字段的实体（如密码本条目需要支持私有/公有）：

```csharp
// 实体定义
public class PasswordBookEntry : MultiTenantEntity
{
    /// <summary>拥有者用户ID（用于加密密钥派生）</summary>
    public string UserId { get; set; } = string.Empty;

    /// <summary>是否公有（true: 企业内可见, false: 仅自己可见）</summary>
    public bool IsPublic { get; set; } = false;
}
```

服务层过滤：
```csharp
// 查询可见数据：自己的私有 + 企业内公有
var items = await _factory.FindAsync(
    filter: e => e.IsPublic || e.UserId == currentUserId);
```
