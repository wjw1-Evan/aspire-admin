# 中间件与响应

## 中间件顺序（固定）

中间件必须按以下顺序注册，**禁止随意调整顺序**：

```
UseExceptionHandler  →  UseCors  →  UseAuthentication  →  UseAuthorization  →  UseMiddleware<ActivityLogMiddleware>  →  UseMiddleware<ResponseFormattingMiddleware>  →  MapControllers
```

| 序号 | 中间件 | 职责 |
|------|--------|------|
| 1 | ExceptionHandler | 全局异常处理 |
| 2 | CORS | 跨域资源共享 |
| 3 | Authentication | 用户认证 |
| 4 | Authorization | 授权检查 |
| 5 | ActivityLogMiddleware | HTTP 访问日志 |
| 6 | ResponseFormattingMiddleware | 统一响应格式 |
| 7 | MapControllers | 路由映射 |

### 示例代码
```csharp
// Program.cs
var builder = WebApplication.CreateBuilder(args);

// 中间件顺序固定
if (app.Environment.IsDevelopment())
{
    app.UseExceptionHandler();
}
else
{
    app.UseExceptionHandler("/Error");
}

app.UseCors();
app.UseAuthentication();
app.UseAuthorization();
app.UseMiddleware<ActivityLogMiddleware>();      // 活动日志
app.UseMiddleware<ResponseFormattingMiddleware>(); // 响应格式

app.MapControllers();
```

## 统一响应格式

### ResponseFormattingMiddleware

#### 职责
- 统一所有 API 的 JSON 响应格式。
- 自动包裹为 `success` / `data` / `timestamp` 结构。

#### 工作方式
- 拦截响应流，若 `Content-Type` 为 `application/json` 且 `StatusCode == 200`：
  - 若响应体**尚未**包含 `"success"` 字段，则将原始 JSON 反序列化为 `data`，再包裹为：
    ```json
    {
      "success": true,
      "data": <原始对象>,
      "timestamp": "2026-03-22T10:00:00Z"
    }
    ```
  - 若已是标准格式（包含 `"success"`），则直接透传。

#### 强制启用
- 所有 HTTP API 服务必须启用 `ResponseFormattingMiddleware`。
- 仅对健康检查、OpenAPI、SSE 等特殊路径做白名单跳过：
  - `/health*`
  - `/scalar*`
  - `/openapi*`
  - `/api/chat/sse`

### JSON 序列化配置
```csharp
// camelCase 命名策略
// 忽略 null 字段
// 枚举序列化为字符串
services.AddControllers()
    .AddJsonOptions(options =>
    {
        options.JsonSerializerOptions.PropertyNamingPolicy = 
            System.Text.Json.JsonNamingPolicy.CamelCase;
        options.JsonSerializerOptions.DefaultIgnoreCondition = 
            System.Text.Json.Serialization.JsonIgnoreCondition.WhenWritingNull;
    });
```

## ActivityLogMiddleware

### 职责
- 记录所有重要 HTTP 请求的活动日志。
- 写入审计日志集合（用户、企业、路径、耗时、状态码、响应摘要等）。

### 配置选项
```csharp
// appsettings.json
{
  "ActivityLog": {
    "Enabled": true,
    "IncludeAnonymous": false,
    "MaxQueryStringLength": 500,
    "IncludeQueryString": true,
    "ExcludedPaths": ["/health", "/metrics"]
  }
}
```

### 日志字段
- 用户信息：`userId`、`username`
- 请求信息：`httpMethod`、`path`、`queryString`、`scheme`、`host`
- 响应信息：`statusCode`、`durationMs`、`responseBody`
- 环境信息：客户端 IP、User-Agent

## SSE 端点处理

### 跳过响应格式化
SSE（Server-Sent Events）端点返回 `text/event-stream`，需跳过 `ResponseFormattingMiddleware` 格式化：

```csharp
// 在 ResponseFormattingMiddleware 中排除 SSE 路径
var path = context.Request.Path.Value;
if (path?.StartsWith("/api/chat/sse", StringComparison.OrdinalIgnoreCase) == true)
{
    await _next(context);
    return;
}
```

### SSE 事件格式
```csharp
// event: {eventType}\ndata: {jsonData}\n\n
// 示例：
// event: Connected\ndata: {"sessionId":"xxx"}\n\n
// event: Message\ndata: {"content":"hello"}\n\n
```

## 错误处理

### 统一错误码
| 错误码 | 说明 | HTTP 状态码 |
|--------|------|-------------|
| `VALIDATION_ERROR` | 参数/模型验证错误 | 400 |
| `UNAUTHORIZED` | 未授权（未登录） | 401 |
| `FORBIDDEN` | 禁止访问（无权限） | 403 |
| `NOT_FOUND` | 资源不存在 | 404 |
| `INTERNAL_ERROR` | 服务器内部错误 | 500 |

### 错误处理原则
- **禁止**暴露底层异常信息（如堆栈、数据库错误）。
- 记录详细错误日志供运维排查。
- 返回给前端的错误消息要可读、可本地化。

```csharp
// ✅ 正确：统一错误码
return BadRequest(new { errorCode = "VALIDATION_ERROR", message = "ID 不能为空" });

// ❌ 禁止：暴露异常详情
return BadRequest(new { error = ex.Message, stackTrace = ex.StackTrace });
```
