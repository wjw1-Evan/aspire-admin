# 中间件与响应

## 中间件顺序（固定）

中间件必须按以下顺序注册，**禁止随意调整顺序**：

```
UseExceptionHandler  →  UseCors  →  UseAuthentication  →  UseAuthorization  →  UseMiddleware<ActivityLogMiddleware>  →  UseMiddleware<ResponseFormattingMiddleware>  →  MapControllers
```

| 序号 | 中间件 | 职责 |
|------|--------|------|
| 1 | ExceptionHandler | 全局异常处理 |
| 2 | CORS | 跨域资源共享 |
| 3 | Authentication | 用户认证 |
| 4 | Authorization | 授权检查 |
| 5 | ActivityLogMiddleware | HTTP 访问日志 |
| 6 | ResponseFormattingMiddleware | 统一响应格式 |
| 7 | MapControllers | 路由映射 |

### 示例代码
```csharp
// Program.cs
var builder = WebApplication.CreateBuilder(args);

// 中间件顺序固定
if (app.Environment.IsDevelopment())
{
    app.UseExceptionHandler();
}
else
{
    app.UseExceptionHandler("/Error");
}

app.UseCors();
app.UseAuthentication();
app.UseAuthorization();
app.UseMiddleware<ActivityLogMiddleware>();      // 活动日志
app.UseMiddleware<ResponseFormattingMiddleware>(); // 响应格式

app.MapControllers();
```

## 统一响应格式

### ResponseFormattingMiddleware

#### 职责
- 统一所有 API 的 JSON 响应格式。
- 自动包裹为 `success` / `data` / `timestamp` 结构。

#### 工作方式
- 拦截响应流，若 `Content-Type` 为 `application/json` 且 `StatusCode == 200`：
  - 若响应体**尚未**包含 `"success"` 字段，则将原始 JSON 反序列化为 `data`，再包裹为：
    ```json
    {
      "success": true,
      "data": <原始对象>,
      "timestamp": "2026-03-22T10:00:00Z"
    }
    ```
  - 若已是标准格式（包含 `"success"`），则直接透传。

#### 强制启用
- 所有 HTTP API 服务必须启用 `ResponseFormattingMiddleware`。
- 仅对健康检查、OpenAPI、SSE 等特殊路径做白名单跳过：
  - `/health*`
  - `/scalar*`
  - `/openapi*`
  - `/api/chat/sse`

### JSON 序列化配置
```csharp
// camelCase 命名策略
// 忽略 null 字段
// 枚举序列化为字符串
services.AddControllers()
    .AddJsonOptions(options =>
    {
        options.JsonSerializerOptions.PropertyNamingPolicy = 
            System.Text.Json.JsonNamingPolicy.CamelCase;
        options.JsonSerializerOptions.DefaultIgnoreCondition = 
            System.Text.Json.Serialization.JsonIgnoreCondition.WhenWritingNull;
    });
```

## ActivityLogMiddleware

### 职责
- 记录所有重要 HTTP 请求的活动日志。
- 写入审计日志集合（用户、企业、路径、耗时、状态码、响应摘要等）。

### 配置选项
```csharp
// appsettings.json
{
  "ActivityLog": {
    "Enabled": true,
    "IncludeAnonymous": false,
    "MaxQueryStringLength": 500,
    "IncludeQueryString": true,
    "ExcludedPaths": ["/health", "/metrics"]
  }
}
```

### 日志字段
- 用户信息：`userId`、`username`
- 请求信息：`httpMethod`、`path`、`queryString`、`scheme`、`host`
- 响应信息：`statusCode`、`durationMs`、`responseBody`
- 环境信息：客户端 IP、User-Agent

## SSE 端点处理

### 跳过响应格式化
SSE（Server-Sent Events）端点返回 `text/event-stream`，需跳过 `ResponseFormattingMiddleware` 格式化：

```csharp
// 在 ResponseFormattingMiddleware 中排除 SSE 路径
var path = context.Request.Path.Value;
if (path?.StartsWith("/api/chat/sse", StringComparison.OrdinalIgnoreCase) == true)
{
    await _next(context);
    return;
}
```

### SSE 事件格式
```csharp
// event: {eventType}\ndata: {jsonData}\n\n
// 示例：
// event: Connected\ndata: {"sessionId":"xxx"}\n\n
// event: Message\ndata: {"content":"hello"}\n\n
```

## BaseApiController 响应方法

### 方法签名

| 方法 | 用途 |
|------|------|
| `Success(object data, string? message = null)` | 返回成功响应 |
| `Fail(string message, int httpStatusCode = 400)` | 返回失败响应 |
| `ValidateModelState()` | 验证模型状态，返回错误或 null |
| `GetCompanyId(bool required = false)` | 获取当前企业 ID |
| `GetClientIpAddress()` | 获取客户端真实 IP |

### Fail 方法状态码

```csharp
// 调用示例
Fail("参数错误")                    // HTTP 400
Fail("未授权访问", 401)             // HTTP 401
Fail("禁止访问", 403)               // HTTP 403
Fail("资源不存在", 404)             // HTTP 404
Fail("服务器异常", 500)             // HTTP 500
```

### 用户上下文

```csharp
// 获取当前用户 ID
var userId = CurrentUserId;

// 获取当前用户 ID（必填，未获取则抛异常）
var userId = CurrentUserId ?? throw new UnauthorizedAccessException("未找到用户信息");

// 获取当前企业 ID（可选）
var companyId = await GetCompanyId();

// 获取当前企业 ID（必填，未获取则抛异常）
var companyId = await GetCompanyId(true);
```

### 使用规范

#### 1. 返回数据（必选）
```csharp
// ✅ 正确：返回具体数据
return Success(user);
return Success(new { id, name, email });
return Success(result);
return Success(pagedResult);

// ❌ 错误：data 不能为 null（必选参数）
return Success();
```

#### 2. 返回操作结果 + 消息
```csharp
// ✅ 正确：data 为 null，消息在第二个参数
return Success(null, "用户创建成功");
return Success(null, "审批已通过");
return Success(null, ErrorMessages.DeleteSuccess);
```

#### 3. 分页响应
```csharp
// ✅ 正确：直接返回 PagedResult<T>
return Success(pagedResult);
```

## 错误处理

### 统一错误码
| 错误码 | 说明 |
|--------|------|
| `VALIDATION_ERROR` | 参数/模型验证错误 |
| `UNAUTHORIZED` | 未授权（未登录） |
| `FORBIDDEN` | 禁止访问（无权限） |
| `NOT_FOUND` | 资源不存在 |
| `INTERNAL_ERROR` | 服务器内部错误 |

### 错误处理原则
- **禁止**暴露底层异常信息（如堆栈、数据库错误）。
- 记录详细错误日志供运维排查。
- 返回给前端的错误消息要可读、可本地化。
- 使用 `Fail()` 方法统一返回错误，手动指定 HTTP 状态码。

```csharp
// ✅ 正确：使用 Fail 方法
return Fail("ID 不能为空");
return Fail($"任务 {taskId} 不存在", 404);

// ❌ 禁止：直接返回原始异常
return BadRequest(new { error = ex.Message, stackTrace = ex.StackTrace });
```

## 错误处理

### 统一错误码
| 错误码 | 说明 |
|--------|------|
| `VALIDATION_ERROR` | 参数/模型验证错误 |
| `UNAUTHORIZED` | 未授权（未登录） |
| `FORBIDDEN` | 禁止访问（无权限） |
| `NOT_FOUND` | 资源不存在 |
| `INTERNAL_ERROR` | 服务器内部错误 |

### 错误处理原则
- **禁止**暴露底层异常信息（如堆栈、数据库错误）。
- 记录详细错误日志供运维排查。
- 返回给前端的错误消息要可读、可本地化。
- 使用标准异常 (`ArgumentException` / `InvalidOperationException`) 抛出错误，由 `BusinessExceptionFilter` 统一处理。

### 标准异常使用

```csharp
// 参数验证错误 -> HTTP 400
throw new ArgumentException("用户名不能为空", "username");
throw new ArgumentException("密码格式不正确", "password");

// 业务逻辑错误 -> HTTP 400
throw new InvalidOperationException("用户名或密码错误");
throw new InvalidOperationException("验证码无效");
throw new InvalidOperationException("用户不存在");

// 资源不存在 -> HTTP 404
throw new InvalidOperationException("任务 (ID: xxx) 不存在");
throw new InvalidOperationException("企业不存在");

// 未授权 -> HTTP 401
throw new UnauthorizedAccessException("未授权访问");
```

### 异常过滤器处理逻辑

```csharp
// BusinessExceptionFilter 会自动处理以下异常类型：
// - ArgumentException / InvalidOperationException -> HTTP 400
// - UnauthorizedAccessException -> HTTP 401
```

### 控制器处理

```csharp
// ✅ 正确：Service 层直接返回实体，异常由过滤器处理
var user = await _userService.GetByIdAsync(id);
return Success(user);

// ✅ 正确：Service 层抛标准异常，Controller 无需处理
await _authService.ChangePasswordAsync(request);
return Success(true, "密码修改成功");

// ✅ 正确：直接返回数据
var users = await _userService.GetAllAsync();
return Success(users);
```
