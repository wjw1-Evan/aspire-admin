# 分页处理规范

本规范结合 AGENTS.md 最高准则，统一所有分页 API 的实现方式。

## 1. 分页类型定义

项目使用 `System.Linq.Dynamic.Core.PagedResult<T>` 作为统一分页返回类型：

```csharp
using System.Linq.Dynamic.Core;
```

### PagedResult<T> 结构

| 属性 | 类型 | 说明 |
|------|------|------|
| Queryable | IQueryable<T> | 分页后的查询对象，需调用 `ToListAsync()` 获取数据 |
| CurrentPage | int | 当前页码 |
| PageSize | int | 每页数量 |
| RowCount | int | 总行数 |
| PageCount | int | 总页数 |

---

## 2. 服务层实现

### 标准分页方法

服务层使用 `PageResult()` 扩展方法返回分页结果：

```csharp
using System.Linq.Dynamic.Core;

public async Task<PagedResult<FileItem>> GetFileItemsAsync(FileListQuery query)
{
    var q = _context.Set<FileItem>().Where(x => x.Status == FileStatus.Active);
    if (query.Type.HasValue)
        q = q.Where(x => x.Type == query.Value);

    // 动态排序（必须先排序）
    q = query.SortBy?.ToLower() switch
    {
        "name" => query.SortOrder == "desc" ? q.OrderByDescending(f => f.Name) : q.OrderBy(f => f.Name),
        "size" => query.SortOrder == "desc" ? q.OrderByDescending(f => f.Size) : q.OrderBy(f => f.Size),
        "createdat" => query.SortOrder == "desc" ? q.OrderByDescending(f => f.CreatedAt) : q.OrderBy(f => f.CreatedAt),
        _ => q.OrderBy(f => f.Name)
    };

    return q.PageResult(query.Page, query.PageSize);
}
```

### 同时提供 List 和 Paginated 版本

如果需要同时提供列表和分页版本，使用 `*PaginatedAsync` 命名约定：

```csharp
// 接口定义
Task<List<FileVersion>> GetVersionHistoryAsync(string fileItemId);
Task<PagedResult<FileVersion>> GetVersionHistoryPaginatedAsync(string fileItemId, int page, int pageSize);

// 实现
public async Task<List<FileVersion>> GetVersionHistoryAsync(string fileItemId)
    => await _context.Set<FileVersion>()
        .Where(v => v.FileItemId == fileItemId)
        .OrderByDescending(v => v.VersionNumber)
        .ToListAsync();

public Task<PagedResult<FileVersion>> GetVersionHistoryPaginatedAsync(string fileItemId, int page, int pageSize)
{
    var query = _context.Set<FileVersion>()
        .Where(v => v.FileItemId == fileItemId)
        .OrderByDescending(v => v.VersionNumber);
    return Task.FromResult(query.PageResult(page, pageSize));
}
```

## 3. 控制器层

控制器直接调用服务层分页方法，**不应在控制器层处理分页逻辑**：

```csharp
[HttpGet("versions")]
public async Task<IActionResult> GetVersionHistory(
    [FromQuery] string fileId,
    [FromQuery] int page = 1,
    [FromQuery] int pageSize = 50)
{
    var pagedResult = await _fileVersionService.GetVersionHistoryPaginatedAsync(fileId, page, pageSize);
    return Success(pagedResult);
}
```

### 禁止在控制器中处理分页

```csharp
// ❌ 禁止：控制器中手动处理分页
[HttpGet("versions")]
public async Task<IActionResult> GetVersionHistory(...)
{
    var versions = await _service.GetVersionHistoryAsync(fileId);
    var ordered = versions.OrderByDescending(v => v.VersionNumber).AsQueryable();
    var pagedResult = ordered.PageResult(page, pageSize);  // 分页逻辑应在服务层
    return Success(pagedResult);
}
```

## 4. 统一分页参数命名

所有分页 API 统一使用以下参数命名：

| 参数 | 说明 | 禁止使用 |
|------|------|----------|
| `page` | 当前页码（从 1 开始） | `current`、`pageIndex`、`pageNumber` |
| `pageSize` | 每页数量 | `limit`、`size`、`perPage` |

### 参数验证

**无需手动验证**：DynamicLinq 的 `PageResult` 会自动处理边界情况（如 `page < 1` 时返回第 1 页）。

---

## 5. 必须先排序

使用 `PageResult` 前，查询**必须先排序**，否则会抛异常：

```csharp
// ✅ 正确：先排序再分页
var result = query.OrderByDescending(x => x.CreatedAt).PageResult(page, pageSize);

// ❌ 错误：未排序直接分页
var result = query.PageResult(page, pageSize);  // 会抛异常
```

## 7. 常见错误示例

```csharp
// ❌ 错误 1：手动分页（绕过 DynamicLinq）
var skip = (page - 1) * pageSize;
var items = await q.Skip(skip).Take(pageSize).ToListAsync();

// ❌ 错误 2：先 ToListAsync 再分页（内存分页）
var all = await q.ToListAsync();
return all.Skip((page - 1) * pageSize).Take(pageSize).ToList();

// ❌ 错误 3：控制器中处理分页
var versions = await _service.GetVersionHistoryAsync(fileId);
var pagedResult = versions.AsQueryable().PageResult(page, pageSize);  // 应该在服务层

// ❌ 错误 4：未排序直接分页
var result = query.PageResult(page, pageSize);  // 会抛异常

// ❌ 错误 5：使用完整命名空间
public async Task<System.Linq.Dynamic.Core.PagedResult<FileItem>> GetFileItemsAsync(...)
```

## 9. 前端对接

前端应使用 `@/types/unified-api` 中的统一类型：

```typescript
import type { ApiResponse, PagedResult } from '@/types/unified-api';

// API 返回类型
type FileListResponse = ApiResponse<PagedResult<FileItem>>;

// 使用示例
const response = await getFileList(params);
const { queryable, rowCount, pageCount } = response.data;
```
