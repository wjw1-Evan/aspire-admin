# 分页处理规范

> **[强制]** 所有分页相关 API 必须遵循以下规范。

---

## DynamicLinq 分页（统一方式）

> **[强制]** 所有分页查询必须使用 `Microsoft.EntityFrameworkCore.DynamicLinq` 提供的 `PageResult` 扩展方法。

### NuGet 依赖
项目已引入 `Microsoft.EntityFrameworkCore.DynamicLinq` 包：
```xml
<PackageReference Include="Microsoft.EntityFrameworkCore.DynamicLinq" Version="10.7.1" />
```

### Using 语句
```csharp
using System.Linq.Dynamic.Core;
```

### 核心扩展方法
| 方法 | 说明 |
|------|------|
| `PageResult(page, pageSize)` | 返回 `System.Linq.Dynamic.Core.PagedResult<T>`，包含 `Queryable`、`CurrentPage`、`PageSize`、`RowCount`、`PageCount` |

### 标准分页查询模式
```csharp
using System.Linq.Dynamic.Core;

// ✅ 正确：使用 DynamicLinq 的 PageResult
public async Task<System.Linq.Dynamic.Core.PagedResult<FileItem>> GetFileItemsAsync(FileListQuery query)
{
    var q = _context.Set<FileItem>().Where(x => x.Status == FileStatus.Active);

    if (query.Type.HasValue)
        q = q.Where(x => x.Type == query.Type.Value);

    // 动态排序
    q = query.SortBy?.ToLower() switch
    {
        "name" => query.SortOrder == "desc" ? q.OrderByDescending(f => f.Name) : q.OrderBy(f => f.Name),
        "size" => query.SortOrder == "desc" ? q.OrderByDescending(f => f.Size) : q.OrderBy(f => f.Size),
        "createdat" => query.SortOrder == "desc" ? q.OrderByDescending(f => f.CreatedAt) : q.OrderBy(f => f.CreatedAt),
        _ => q.OrderBy(f => f.Name)
    };

    return q.PageResult(query.Page, query.PageSize);
}
```

### 控制器层调用
```csharp
[HttpGet("items")]
public async Task<IActionResult> GetFileItems([FromQuery] FileListQuery query)
{
    var result = await _cloudStorageService.GetFileItemsAsync(parentId, query);
    return SuccessPaged(await result.Queryable.ToListAsync(), result.RowCount, result.CurrentPage, result.PageSize);
}
```

### PagedResult<T> 结构
`System.Linq.Dynamic.Core.PagedResult<T>` 包含以下属性：

| 属性 | 类型 | 说明 |
|------|------|------|
| `Queryable` | `IQueryable<T>` | 分页后的查询对象，需调用 `ToListAsync()` 获取数据 |
| `CurrentPage` | `int` | 当前页码 |
| `PageSize` | `int` | 每页大小 |
| `RowCount` | `long` | 总记录数 |
| `PageCount` | `int` | 总页数 |

### ⚠️ 重要：必须先排序
使用 `PageResult` 前，查询必须先排序（`OrderBy`），否则会抛异常：

```csharp
// ✅ 正确：先排序再分页
var result = query.OrderByDescending(x => x.CreatedAt).PageResult(page, pageSize);

// ❌ 错误：未排序直接分页
var result = query.PageResult(page, pageSize);  // 会抛异常
```

## 统一分页参数

### API 参数命名
所有分页 API 统一使用 `page` 和 `pageSize` 作为查询参数：

```csharp
// ✅ 正确：使用 page 和 pageSize
[HttpGet]
public async Task<IActionResult> GetUsers(
    [FromQuery] int page = 1,
    [FromQuery] int pageSize = 20)
{
    // ...
}

// ❌ 禁止：使用其他命名
[HttpGet]
public async Task<IActionResult> GetUsers(
    [FromQuery] int current = 1,
    [FromQuery] int pageIndex = 1)
{
    // ...
}
```

### 参数命名对照表

| 正确命名 | 禁止使用 |
|----------|----------|
| `page` | `current`、`pageIndex`、`pageNumber`、`skip` |
| `pageSize` | `limit`、`size`、`perPage`、`take` |

## 参数验证

### 无需验证
**[强制]** 不要在控制器或服务层手动验证分页参数。DynamicLinq 的 `PageResult` 会自动处理边界情况（如 `page < 1` 时返回第 1 页）。

```csharp
// ✅ 正确：不验证分页参数，直接调用
[HttpGet]
public async Task<IActionResult> GetUsers([FromQuery] int page = 1, [FromQuery] int pageSize = 20)
{
    var result = await _userService.GetUsersAsync(page, pageSize);
    return SuccessPaged(await result.Queryable.ToListAsync(), result.RowCount, result.CurrentPage, result.PageSize);
}

// ❌ 禁止：手动验证分页参数
if (page < 1 || page > 10000)
    return ValidationError("页码必须在 1-10000 之间");
```

## 分页查询

### 直接传递 page 参数
调用 `PageResult` 时，**直接传递 `page` 参数**（页码，从 1 开始），**不要计算 `skip`**：

```csharp
// ✅ 正确：直接传递页码
var pagedResult = q.PageResult(page, pageSize);
var items = await pagedResult.Queryable.ToListAsync();

// ❌ 错误：计算 skip
// var skip = (page - 1) * pageSize;
// var items = await q.Skip(skip).Take(pageSize).ToListAsync();
```

## 标准参考实现

`CloudStorageService.GetFileItemsAsync` 为标准实现，所有分页接口应参考此实现：

```csharp
/// <summary>
/// 获取文件列表（分页）
/// </summary>
public async Task<System.Linq.Dynamic.Core.PagedResult<FileItem>> GetFileItemsAsync(string parentId, FileListQuery query)
{
    // 1. 构建查询
    var q = _context.Set<FileItem>().Where(x => x.Status == FileStatus.Active);

    // 2. 动态过滤
    if (query.Type.HasValue)
        q = q.Where(x => x.Type == query.Type.Value);

    // 3. 动态排序（必须）
    q = query.SortBy?.ToLower() switch
    {
        "name" => query.SortOrder == "desc" ? q.OrderByDescending(f => f.Name) : q.OrderBy(f => f.Name),
        "size" => query.SortOrder == "desc" ? q.OrderByDescending(f => f.Size) : q.OrderBy(f => f.Size),
        "createdat" => query.SortOrder == "desc" ? q.OrderByDescending(f => f.CreatedAt) : q.OrderBy(f => f.CreatedAt),
        _ => q.OrderBy(f => f.Name)
    };

    // 4. 使用 DynamicLinq 分页
    return q.PageResult(query.Page, query.PageSize);
}
```

## 常见错误示例

```csharp
// ❌ 错误 1：手动分页（绕过 DynamicLinq）
var skip = (page - 1) * pageSize;
var items = await q.Skip(skip).Take(pageSize).ToListAsync();
var total = await q.CountAsync();

// ❌ 错误 2：先 ToListAsync 再分页（内存分页，性能极差）
var all = await q.ToListAsync();
return all.Skip((page - 1) * pageSize).Take(pageSize).ToList();

// ❌ 错误 3：手动验证分页参数
if (page < 1 || page > 10000)
    return ValidationError("页码必须在 1-10000 之间");

// ❌ 错误 4：响应格式不统一
return Ok(new { items, total });  // 缺少 page 和 pageSize

// ❌ 错误 5：未排序直接分页
var result = query.PageResult(page, pageSize);  // 会抛异常

// ❌ 错误 6：使用自定义 PagedResult
public async Task<PagedResult<FileItem>> GetFileItemsAsync(...)  // 应使用 DynamicLinq 的 PagedResult
```
