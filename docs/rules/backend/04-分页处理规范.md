# 分页处理规范

> **[强制]** 所有分页相关 API 必须遵循以下规范。

## 统一分页参数

### API 参数命名
所有分页 API 统一使用 `page` 和 `pageSize` 作为查询参数：

```csharp
// ✅ 正确：使用 page 和 pageSize
[HttpGet]
public async Task<IActionResult> GetUsers(
    [FromQuery] int page = 1,
    [FromQuery] int pageSize = 20)
{
    // ...
}

// ❌ 禁止：使用其他命名
[HttpGet]
public async Task<IActionResult> GetUsers(
    [FromQuery] int current = 1,
    [FromQuery] int pageIndex = 1)
{
    // ...
}
```

### 参数命名对照表

| 正确命名 | 禁止使用 |
|----------|----------|
| `page` | `current`、`pageIndex`、`pageNumber`、`skip` |
| `pageSize` | `limit`、`size`、`perPage`、`take` |

## 参数验证

### 验证规则
- `page`：必须在 1-10000 之间
- `pageSize`：必须在 1-100 之间
- 超出范围抛出 `ArgumentException`

### 代码示例
```csharp
// ✅ 正确：统一使用 ArgumentException
if (page < 1 || page > 10000)
    throw new ArgumentException("页码必须在 1-10000 之间");

if (pageSize < 1 || pageSize > 100)
    throw new ArgumentException("每页数量必须在 1-100 之间");

// ❌ 禁止：静默修正参数值
if (page < 1) page = 1;  // 不要静默修正
```

### POST 请求分页
如果使用 POST 请求传递分页参数（如 `QueryRequest`），请求体中的分页参数仍使用 `Current` 和 `PageSize`：

```csharp
public class UserQueryRequest
{
    public int Current { get; set; } = 1;
    public int PageSize { get; set; } = 20;
    // 其他查询参数...
}

[HttpPost("list")]
public async Task<IActionResult> GetUsers([FromBody] UserQueryRequest request)
{
    // 验证分页参数
    if (request.Current < 1 || request.Current > 10000)
        throw new ArgumentException("页码必须在 1-10000 之间");

    if (request.PageSize < 1 || request.PageSize > 100)
        throw new ArgumentException("每页数量必须在 1-100 之间");

    // ...
}
```

## 分页查询

### 直接传递 page 参数
调用 `FindPagedAsync` 时，**直接传递 `page` 参数**（页码，从 1 开始），**不要计算 `skip`**：

```csharp
// ✅ 正确：直接传递页码
var (users, total) = await _userFactory.FindPagedAsync(
    filter,
    sortBuilder.Build(),
    page,      // 页码，从 1 开始
    pageSize
);

// ❌ 错误：计算 skip
// var skip = (page - 1) * pageSize;
// var (users, total) = await _userFactory.FindPagedAsync(filter, sortBuilder.Build(), skip, pageSize);
```

## 分页响应格式

### 统一响应格式
```csharp
// ✅ 正确：标准分页响应
return Success(new
{
    data = items,
    total = total,
    page = page,
    pageSize = pageSize
});

// ✅ 正确：使用扩展方法
return Success((items, total).ToPaginatedResponse(page, pageSize));
```

### 响应结构
```json
{
  "success": true,
  "data": {
    "data": [...],
    "total": 100,
    "page": 1,
    "pageSize": 20
  },
  "timestamp": "2026-03-22T10:00:00Z"
}
```

## 标准参考实现

`UserController.GetCurrentUserActivityLogsPaged` 为标准实现，所有分页接口应参考此实现：

```csharp
/// <summary>
/// 获取当前用户的活动日志（分页）
/// </summary>
/// <param name="page">页码（默认1）</param>
/// <param name="pageSize">每页大小（默认20）</param>
[HttpGet("me/activity-logs-paged")]
public async Task<IActionResult> GetCurrentUserActivityLogsPaged(
    [FromQuery] int page = 1,
    [FromQuery] int pageSize = 20)
{
    // 1. 验证参数
    if (page < 1 || page > 10000)
        throw new ArgumentException("页码必须在 1-10000 之间");

    if (pageSize < 1 || pageSize > 100)
        throw new ArgumentException("每页数量必须在 1-100 之间");

    // 2. 获取当前用户
    var userId = GetRequiredUserId();

    // 3. 查询数据
    var (logs, total) = await _userActivityLogService.GetPagedLogsAsync(userId, page, pageSize);

    // 4. 返回分页响应
    return Success((logs, total).ToPaginatedResponse(page, pageSize));
}
```

## 常见错误示例

```csharp
// ❌ 错误 1：参数验证不统一
[HttpGet]
public async Task<IActionResult> GetItems([FromQuery] int pageIndex)
{
    // pageIndex 不是标准命名
}

// ❌ 错误 2：静默修正参数
[HttpGet]
public async Task<IActionResult> GetItems([FromQuery] int page)
{
    if (page < 1) page = 1;  // 不要静默修正
}

// ❌ 错误 3：手动计算 skip
var skip = (page - 1) * pageSize;
var items = await _factory.FindPagedAsync(filter, sort, skip, pageSize);

// ❌ 错误 4：响应格式不统一
return Ok(new { items, total });  // 缺少 page 和 pageSize
```
