# 批量查询规范（N+1 防护）

> **[强制]** 严禁在循环内调用单条查询，必须使用批量查询方法。

## N+1 查询问题

### 问题描述
在循环内调用单条查询方法会导致 N+1 查询问题，严重影响性能：

```csharp
// ❌ 错误：N+1 查询
foreach (var task in tasks)
{
    var user = await _userService.GetUserByIdAsync(task.CreatedBy);  // N 次查询
    dto.CreatedByName = user.Username;
}
```

### 性能影响
- 100 个任务 → 100 次数据库查询
- 预期：1 次批量查询

## 批量查询接口

### IUserService 必须提供
```csharp
public interface IUserService
{
    /// <summary>
    /// 根据ID列表批量获取用户（带租户过滤）
    /// </summary>
    /// <param name="ids">用户ID列表</param>
    /// <returns>用户字典，Key为用户ID</returns>
    Task<Dictionary<string, User>> GetUsersByIdsAsync(IEnumerable<string> ids);
}
```

### 实现示例
```csharp
public async Task<Dictionary<string, User>> GetUsersByIdsAsync(IEnumerable<string> ids)
{
    var idList = ids.Where(id => !string.IsNullOrEmpty(id)).Distinct().ToList();
    if (idList.Count == 0)
    {
        return new Dictionary<string, User>();
    }

    var users = await _userFactory.FindAsync(u => idList.Contains(u.Id));
    return users.ToDictionary(u => u.Id!, u => u);
}
```

## 批量转换 DTO 模式

### 标准模式（4 步）
1. **收集 ID**：使用 `HashSet<string>` 去重，一次性收集所有需要查询的用户 ID
2. **批量查询**：调用 `GetUsersByIdsAsync` 一次性获取所有数据
3. **内存映射**：使用 `Dictionary` 在内存中进行映射
4. **禁止循环查询**：在循环内调用单条查询

### 代码示例
```csharp
private async Task<List<TaskDto>> ConvertToTaskDtosAsync(IEnumerable<WorkTask> tasks)
{
    var taskList = tasks.ToList();
    if (taskList.Count == 0)
    {
        return new List<TaskDto>();
    }

    // 1️⃣ 收集用户 ID
    var userIds = new HashSet<string>();
    var projectIds = new HashSet<string>();

    foreach (var task in taskList)
    {
        if (!string.IsNullOrEmpty(task.CreatedBy)) userIds.Add(task.CreatedBy);
        if (!string.IsNullOrEmpty(task.AssignedTo)) userIds.Add(task.AssignedTo);
        
        // 参与者
        if (task.ParticipantIds != null)
        {
            foreach (var pid in task.ParticipantIds)
            {
                if (!string.IsNullOrEmpty(pid)) userIds.Add(pid);
            }
        }
        
        // 项目 ID
        if (!string.IsNullOrEmpty(task.ProjectId)) projectIds.Add(task.ProjectId);
    }

    // 2️⃣ 批量查询
    var userMap = await _userService.GetUsersByIdsAsync(userIds);
    var projectMap = await GetProjectNamesAsync(projectIds);

    // 3️⃣ 内存映射转换
    return taskList.Select(t => ConvertToTaskDtoWithCache(t, userMap, projectMap)).ToList();
}

private TaskDto ConvertToTaskDtoWithCache(
    WorkTask task, 
    Dictionary<string, User> userMap,
    Dictionary<string, string> projectMap)
{
    var dto = new TaskDto { /* 基本字段 */ };

    // 4️⃣ 使用 Dictionary 查找，避免 N+1
    if (!string.IsNullOrEmpty(task.CreatedBy) && 
        userMap.TryGetValue(task.CreatedBy, out var createdByUser))
    {
        dto.CreatedByName = !string.IsNullOrWhiteSpace(createdByUser.Name)
            ? $"{createdByUser.Username} ({createdByUser.Name})"
            : createdByUser.Username;
    }

    if (!string.IsNullOrEmpty(task.ProjectId) && 
        projectMap.TryGetValue(task.ProjectId, out var projectName))
    {
        dto.ProjectName = projectName;
    }

    return dto;
}
```

## 常见错误示例

### ❌ 错误 1：循环内单条查询
```csharp
// 100 个任务 = 100 次数据库查询
foreach (var task in tasks)
{
    var user = await _userService.GetUserByIdAsync(task.CreatedBy);
    dto.CreatedByName = user?.Username;
}
```

### ❌ 错误 2：循环内多次查询
```csharp
// 循环内调用多个单条查询
foreach (var task in tasks)
{
    var createdBy = await _userService.GetUserByIdAsync(task.CreatedBy);
    var assignedTo = await _userService.GetUserByIdAsync(task.AssignedTo);
    
    foreach (var pid in task.ParticipantIds)
    {
        var participant = await _userService.GetUserByIdAsync(pid);  // 嵌套循环！
    }
}
```

### ✅ 正确：批量查询
```csharp
// 1 次批量查询
var allUserIds = tasks
    .SelectMany(t => new[] { t.CreatedBy, t.AssignedTo }
        .Concat(t.ParticipantIds ?? Enumerable.Empty<string>()))
    .Where(id => !string.IsNullOrEmpty(id))
    .Distinct();

var userMap = await _userService.GetUsersByIdsAsync(allUserIds);

return tasks.Select(t => ConvertWithCache(t, userMap)).ToList();
```

## 适用场景

### 必须使用批量查询
- 列表页 DTO 转换
- 树形结构数据加载
- 批量导出/导入
- 统计数据的用户信息填充

### 可使用单条查询
- 详情页（确定只有一条数据）
- 更新后的即时返回
- 独立的单次操作
