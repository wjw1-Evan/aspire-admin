# 类型命名规范

> **[强制]** 禁止使用 `using Xxx = Yyy.Zzz` 别名写法，必须直接使用完整类型名。

## 禁用类型别名

### 问题
使用类型别名会导致：
- 类型来源不明确
- 代码可读性降低
- IDE 提示不准确
- 重构困难

### ❌ 禁止写法
```csharp
// 禁止：使用类型别名
using TaskModel = Platform.ApiService.Models.WorkTask;
using TaskStatusEnum = Platform.ApiService.Models.TaskStatus;
using User = Platform.ApiService.Models.AppUser;

public class TaskService
{
    // 别名使用
    private async Task<List<TaskModel>> GetTasksAsync()
    {
        if (t.Status == TaskStatusEnum.Pending) { }  // 别名
    }
}
```

### ✅ 正确写法

#### 直接使用完整类型名
```csharp
public class TaskService : ITaskService
{
    private readonly IDataFactory<WorkTask> _taskFactory;
    
    public async Task<List<WorkTask>> GetTasksAsync()
    {
        // 直接使用 WorkTask 类型
        var tasks = await _taskFactory.FindAsync(t => 
            t.Status == Models.TaskStatus.Pending);
            
        return tasks;
    }
}
```

#### 使用命名空间导入
```csharp
using Platform.ApiService.Models;  // 导入命名空间

public class TaskService : ITaskService
{
    public async Task<List<WorkTask>> ProcessTasksAsync(List<WorkTask> tasks)
    {
        foreach (var task in tasks)
        {
            if (task.Status == TaskStatus.Pending)  // 直接使用枚举
            {
                // 处理逻辑
            }
        }
    }
}
```

## 常见类型使用规范

### 实体类型
| 类型 | 命名空间 | 正确用法 |
|------|----------|----------|
| `WorkTask` | `Platform.ApiService.Models` | `private readonly IDataFactory<WorkTask>` |
| `User` / `AppUser` | `Platform.ApiService.Models` | `var userMap = new Dictionary<string, AppUser>()` |
| `Project` | `Platform.ApiService.Models` | `private readonly IDataFactory<Project>` |

### 枚举类型
```csharp
// ✅ 正确：使用完整命名空间
if (task.Status == Models.TaskStatus.Pending) { }
if (task.Priority == Models.TaskPriority.High) { }

// ✅ 正确：导入命名空间后直接使用
using Platform.ApiService.Models;
if (task.Status == TaskStatus.Pending) { }
```

### DTO 类型
```csharp
// ✅ 正确：使用 Models 命名空间中的 DTO
using Platform.ApiService.Models;

public TaskDto ConvertToDto(WorkTask task)
{
    return new TaskDto
    {
        Id = task.Id,
        TaskName = task.TaskName,
        Status = (int)task.Status
    };
}
```

## 常见错误示例

### ❌ 错误 1：类型别名混淆
```csharp
using TaskModel = Platform.ApiService.Models.WorkTask;
using AnotherTask = Platform.Other.Models.WorkTask;

public void Process()
{
    var t1 = new TaskModel();   // 哪个命名空间？
    var t2 = new AnotherTask(); // 需要查别名定义
}
```

### ❌ 错误 2：循环别名导致
```csharp
// 在多个文件中定义相同的别名
// File1.cs
using User = Platform.ApiService.Models.AppUser;

// File2.cs  
using User = Platform.ApiService.Models.UserEntity;  // 冲突！

// 导致混淆和 bug
```

### ✅ 正确：明确命名空间
```csharp
// 直接使用完整类型
var userMap = new Dictionary<string, Platform.ApiService.Models.AppUser>();

// 或导入命名空间
using AppUser = Platform.ApiService.Models.AppUser;
var userMap = new Dictionary<string, AppUser>();
```

## 迁移指南

### 查找使用别名的文件
```bash
# 查找所有 using 别名
grep -r "using.*=" Platform.ApiService/Services/ --include="*.cs"
```

### 迁移步骤
1. 找到文件中所有的类型别名
2. 移除 `using Xxx = Yyy.Zzz;` 行
3. 替换所有类型别名引用为完整类型名或命名空间前缀
4. 确保命名空间已导入
5. 编译验证

### 示例迁移
```csharp
// 迁移前
using TaskModel = Platform.ApiService.Models.WorkTask;
using TaskStatusEnum = Platform.ApiService.Models.TaskStatus;

public async Task<List<TaskModel>> GetTasksAsync()
{
    var tasks = await _taskFactory.FindAsync(t => 
        t.Status == TaskStatusEnum.Pending);
}

// 迁移后
using Platform.ApiService.Models;

public async Task<List<WorkTask>> GetTasksAsync()
{
    var tasks = await _taskFactory.FindAsync(t => 
        t.Status == Models.TaskStatus.Pending);
}
```

## IDE 配置建议

### ReSharper / Rider
- 启用 "Use fully qualified type names" 规则
- 禁用 "Use shortened 'using' directives"

### Visual Studio
- 安装 "Treat Warnings as Errors" 插件
- 自定义规则检查别名使用

### EditorConfig
```ini
# .editorconfig
dotnet_styleQualification_for_field = true
dotnet_styleQualification_for_property = true
dotnet_styleQualification_for_type_parameter = true
```
