# 路由与菜单规范

本规范重点强调前端菜单权限与后端 API 权限标识一一对应，确保前后端权限联动。

## 路由与菜单权限配置

### 路由文件

- 路由文件位置：`Platform.Admin/config/routes.ts`
- 使用 Umi 路由配置定义页面路径、组件与菜单元数据。
- **[强制]** 每个菜单/页面的 `name` 字段必须与后端 `[RequireMenu("xxx")]` 权限标识完全一致，实现前后端权限统一。

```typescript
// config/routes.ts
export default [
  {
    path: '/task-management',
    name: 'task-management', // 必须与后端 RequireMenu("task-management") 对应
    icon: 'TaskIcon',
    component: './task-management',
    routes: [
      {
        path: '/task-management',
        redirect: '/task-management/list',
      },
      {
        path: '/task-management/list',
        name: 'task-list', // 对应 RequireMenu("task-list")
        component: './task-management/index',
      },
    ],
  },
];
```

### 路由配置字段

| 字段 | 说明 | 必填 |
|------|------|------|
| `path` | 路由路径 | 是 |
| `name` | 菜单权限标识（唯一，需与后端一致） | 是 |
| `component` | 组件路径 | 是 |
| `icon` | 菜单图标 | 否 |
| `routes` | 子路由 | 否 |
| `redirect` | 重定向 | 否 |

> **注意：** name 字段即为权限标识，前端菜单权限与后端 API 权限完全一致，便于统一鉴权与动态菜单渲染。

## 菜单与多语言

### 菜单翻译文件

各语言菜单名称在 `src/locales/*/menu.ts` 中维护：

```typescript
// src/locales/zh-CN/menu.ts
export default {
  'menu.task-management': '任务管理',
  'menu.task-management.list': '任务列表',
  'menu.task-management.settings': '任务设置',
  // 更多菜单...
};

// src/locales/en-US/menu.ts
export default {
  'menu.task-management': 'Task Management',
  'menu.task-management.list': 'Task List',
  'menu.task-management.settings': 'Task Settings',
};
```

### 命名规范

- 翻译键格式：`menu.模块.资源`（使用 `.` 分隔）
- 路由 name 与翻译键保持一致

## 与后端菜单权限的映射

### 前端权限控制

- 前端路由/菜单 key 应与后端菜单标识保持稳定映射关系。
- **前端不再依赖隐藏按钮/菜单实现权限**，最终访问控制以后端 `RequireMenu` 判定为准。

### 示例

```typescript
// 前端路由 name（使用连字符格式）
name: 'user-management'

// 后端 RequireMenu
[RequireMenu("user-management")]
```

## 新增页面流程

### 1. 添加路由配置

```typescript
// config/routes.ts
{
  path: '/new-module',
  name: 'new-module',
  icon: 'PlusOutlined',
  component: './new-module/index',
}
```

### 2. 添加菜单翻译

```typescript
// src/locales/zh-CN/menu.ts
export default {
  'menu.new-module': '新模块',
};

// src/locales/en-US/menu.ts
export default {
  'menu.new-module': 'New Module',
};
```

### 3. 创建页面组件

- 遵循页面风格统一规范
- 遵循 TypeScript 类型安全规范

## 常见错误示例

### ❌ 错误 1：翻译键不一致

```typescript
// 路由
name: 'user-management'

// 翻译
'menu.users': '用户管理'  // ❌ 键不一致
'menu.user-management': '用户管理'  // ✅ 正确
```

### ❌ 错误 2：缺少多语言

```typescript
// 只添加了中文
export default {
  'menu.new-feature': '新功能',
};

// 缺少英文
// 'menu.new-feature': 'New Feature',  // ❌ 英文缺失
```

### ❌ 错误 3：前端权限判断

```tsx
// ❌ 禁止：前端权限判断
{hasPermission('user:delete') && (
  <Button danger onClick={handleDelete}>删除</Button>
)}

// ✅ 正确：直接显示，后端控制
<Button danger onClick={handleDelete}>删除</Button>
```
