# 服务层封装

## 统一使用 request

### **[强制]** 所有 HTTP 请求必须通过 `@umijs/max` 的 `request` 封装

```tsx
// ✅ 正确：使用统一 request
import { request } from '@umijs/max';

const response = await request<UserResponse>('/api/users', {
  method: 'GET',
  params: { page: 1, pageSize: 20 },
});

// ❌ 禁止：直接 fetch
fetch('/api/users')
  .then(res => res.json())
  .then(data => setUsers(data));
```

### 服务文件结构

- 集中放在 `src/services/` 目录
- 每个领域一个文件

```
src/services/
```

# 服务层封装与 API 对接规范

> 本文档重点统一前端与后端 API 对接的响应结构、分页、鉴权、SSE 实时通信、错误处理等。**统一分页规范请参阅**：[`00-分页规范.md`](../00-分页规范.md)

## 1. 统一使用 request 封装

- **[强制]** 所有 HTTP 请求必须通过 `@umijs/max` 的 `request` 封装，禁止直接 fetch/axios。

  ```tsx
  import { request } from '@umijs/max';
  const response = await request<UserResponse>('/api/users', {
    method: 'GET',
    params: { page: 1, pageSize: 20 },
  });
  ```

## 2. 服务文件结构

- 集中放在 `src/services/` 目录，每个领域一个文件夹，api.ts/types.ts 分离。

## 3. ApiResponse 统一处理

- 后端所有接口返回统一结构：

  ```typescript
  interface ApiResponse<T> {
    success: boolean;
    data: T;
    errorCode?: string;
    errorMessage?: string;
    timestamp: string;
  }
  ```

- 前端所有接口调用必须先判断 `success`，再处理 `data`，错误统一 message 弹窗。

  ```typescript
  const res = await request<ApiResponse<User[]>>('/api/users');
  if (res.success) {
    setUsers(res.data);
  } else {
    message.error(res.errorMessage || '请求失败');
  }
  ```

- 统一在 request 拦截器中处理 401/403/500 等异常，自动跳转登录或弹窗。

## 4. 分页接口对接

> 详细规范请参阅：[`00-分页规范.md`](../00-分页规范.md#5-前端实现规范)

- 后端分页接口统一返回 `PagedResult<T>`：

  ```typescript
  import type { PagedResult } from '@/types/unified-api';

  interface PagedResult<T> {
    queryable: T[];
    currentPage: number;
    pageSize: number;
    rowCount: number;
    pageCount: number;
  }
  ```

- ProTable/Antd Table 对接示例：

  ```tsx
  import type { ApiResponse, PagedResult } from '@/types/unified-api';
  import type { User } from './types';

  const fetchData = async (params: { page: number; pageSize: number }) => {
    const res = await request<ApiResponse<PagedResult<User>>>('/api/users', { params });
    if (res.success && res.data) {
      return {
        data: res.data.queryable,
        success: res.success,
        total: res.data.rowCount,
      };
    }
    return { data: [], success: false, total: 0 };
  };
  ```

## 5. 鉴权与权限

- 路由/菜单 name 字段必须与后端 RequireMenu("xxx") 权限标识一致。
- 动态渲染菜单、按钮权限时，直接对比后端返回的权限列表。
- 统一在 request 拦截器中处理 401/403，自动跳转登录或提示。

## 6. SSE 实时通信

- 后端全部采用 SSE（Server-Sent Events）推送，前端统一使用 EventSource 订阅。

  ```typescript
  const sse = new EventSource('/api/notify/sse');
  sse.onmessage = (event) => {
    const data = JSON.parse(event.data);
    // 处理消息
  };
  sse.onerror = (err) => {
    // 断线重连或提示
  };
  ```

- 建议封装 SSE 连接管理，自动断线重连。

## 7. 统一最佳实践

- 所有接口调用、数据结构、分页、权限、SSE、错误处理等，必须与后端标准保持一致。
- 发现后端接口变更或文档不符时，优先以后端最新标准为准，及时修订前端服务层实现与文档。

## 常见错误示例

### ❌ 错误 1：未使用统一封装

```tsx
// ❌ 禁止：直接 fetch
useEffect(() => {
  fetch('/api/users')
    .then(res => res.json())
    .then(data => setUsers(data));
}, []);
```

### ❌ 错误 2：未处理错误

```tsx
// ❌ 禁止：未检查 success
const response = await getUserList();
setUsers(response.data);  // 可能失败
```

### ❌ 错误 3：硬编码 URL

```tsx
// ❌ 禁止：硬编码基础 URL
axios.post('http://localhost:5000/api/users', data);

// ✅ 正确：使用相对路径
request('/api/users', { method: 'POST', data });
```

### ❌ 错误 4：使用非标准分页参数名

```tsx
// ❌ 禁止：使用 current 而非 page
const response = await getUsers({ current: 1, pageSize: 20 });

// ✅ 正确：使用 page
const response = await getUsers({ page: 1, pageSize: 20 });
```
