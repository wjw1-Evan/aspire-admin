# 页面与组件结构

本规范重点补充前端页面与后端 API 对接时的数据流、接口调用、分页、错误处理等最佳实践。

## 1. 列表页面结构与数据流

```tsx
import { PageContainer } from '@ant-design/pro-components';
import { Card, Row, Col } from 'antd';
import { StatCard } from '@/components';

const MyListPage: React.FC = () => {
  const [data, setData] = useState<User[]>([]);
  const [loading, setLoading] = useState(false);
  const [pagination, setPagination] = useState({ page: 1, pageSize: 20, total: 0 });
  const searchParamsRef = useRef<PageParams>({ page: 1, pageSize: 20, search: '' });

  const fetchData = async () => {
    setLoading(true);
    try {
      const res = await fetchUsers(searchParamsRef.current);
      if (res.success && res.data) {
        setData(res.data.queryable || []);
        setPagination(prev => ({ ...prev, total: res.data.rowCount ?? 0 }));
      }
    } finally {
      setLoading(false);
    }
  };

  return (
    <PageContainer>
      {/* 统计卡片区域 */}
      <Card style={{ marginBottom: 16 }}>
        <Row gutter={[12, 12]}>
          <Col xs={24} sm={12} md={6}>
            <StatCard title="总数" value={pagination.total} icon={<Icon />} />
          </Col>
        </Row>
      </Card>

      {/* 搜索表单 */}
      <SearchBar
        style={{ marginBottom: 16 }}
        onSearch={(params) => {
          searchParamsRef.current = { ...searchParamsRef.current, ...params, page: 1 };
          fetchData();
        }}
      />

      {/* 数据表格 */}
      <Table
        dataSource={data}
        columns={columns}
        rowKey="id"
        loading={loading}
        pagination={{
          current: pagination.page,
          pageSize: pagination.pageSize,
          total: pagination.total,
          onChange: (page, pageSize) => {
            searchParamsRef.current = { ...searchParamsRef.current, page, pageSize };
            fetchData();
          },
        }}
      />
    </PageContainer>
  );
};
```

## 2. 表单页面结构与错误处理

```tsx
import { Modal, Form, message } from 'antd';
import { createUser } from '@/services/user/api';

interface FormModalProps {
  open: boolean;
  onClose: () => void;
  onSuccess: () => void;
}

const FormModal: React.FC<FormModalProps> = ({ open, onClose, onSuccess }) => {
  const [form] = Form.useForm();

  const handleSubmit = async (values: MyFormValues) => {
    const res = await createUser(values);
    if (res.success) {
      message.success('操作成功');
      onSuccess();
    } else {
      message.error(res.errorMessage || '操作失败');
    }
  };

  return (
    <Modal
      title="标题"
      open={open}
      onCancel={onClose}
      onOk={() => form.submit()}
    >
      <Form form={form} onFinish={handleSubmit}>
        {/* 表单项 */}
      </Form>
    </Modal>
  );
};
```

## 3. 统一分页与后端对接

- 所有分页表格组件必须严格对齐后端 PagedResult 结构，参数命名统一为 page/pageSize。
- 分页、排序、筛选参数需与后端接口保持一致。

## 4. 错误处理与用户提示

- 所有接口调用必须判断 res.success，失败时统一 message.error 提示。
- 401/403/500 等异常由 request 拦截器统一处理。

## 5. 变更与维护建议

- 页面与组件如需扩展，优先遵循后端 API 结构和统一数据流。
- 发现实现与本规范或后端标准不符时，优先以本规范为准，及时修订文档与代码。

## 组件拆分

### 拆分原则

- 复用较高的表单、列表操作栏等拆分为独立组件
- 放在 `src/components/` 或 `pages/xxx/components/` 目录
- UI 组件不直接调用后端 services

### 目录结构示例

```
src/pages/task-management/
├── index.tsx              # 主页面
├── components/
│   ├── TaskForm.tsx       # 任务表单组件
│   ├── TaskDetail.tsx     # 任务详情组件
│   ├── TaskExecutionPanel.tsx  # 执行面板
│   └── ProjectView.tsx    # 项目视图
└── types.ts               # 页面类型定义
```

### 组件通信

```tsx
// ✅ 正确：通过 props 传递数据和回调
interface TaskFormProps {

## 原生 Table 使用

### 基本配置

```tsx
<Table
  dataSource={data}
  columns={columns}
  rowKey="id"
  loading={loading}
  onChange={(pagination, _filters, sorter) => {
    searchParamsRef.current = {
      ...searchParamsRef.current,
      page: pagination.current,
      pageSize: pagination.pageSize,
      sortBy: sorter?.field as string,
      sortOrder: sorter?.order === 'ascend' ? 'asc' : sorter?.order === 'descend' ? 'desc' : undefined,
    };
    fetchData();
  }}
  pagination={{
    current: pagination.page,
    pageSize: pagination.pageSize,
    total: pagination.total
  }}
/>
```

## Hooks 使用

### 自定义 Hooks

```tsx
// src/pages/task-management/hooks/useTaskManagement.ts
import { useState, useCallback } from 'react';
import { queryTasks, deleteTask } from '@/services/task/api';
import type { TaskDto, TaskStatistics } from '../types';

export function useTaskManagement() {
  const [loading, setLoading] = useState(false);
  const [statistics, setStatistics] = useState<TaskStatistics | null>(null);

  const fetchTasks = useCallback(async (params) => {
    setLoading(true);
    try {
      const response = await queryTasks(params);
      return response;
    } finally {
      setLoading(false);
    }
  }, []);

  const handleDelete = useCallback(async (taskId: string) => {
    setLoading(true);
    try {
      await deleteTask(taskId);
      return true;
    } finally {
      setLoading(false);
    }
  }, []);

  return {
    loading,
    statistics,
    fetchTasks,
    handleDelete,
  };
}
```

### 使用自定义 Hooks

```tsx
const MyPage: React.FC = () => {
  const { loading, fetchTasks, handleDelete } = useTaskManagement();

  // ...
};
```

## 常见错误示例

### ❌ 错误 1：页面堆积过多逻辑

```tsx
// ❌ 禁止：一个文件超过 500 行
const MyPage: React.FC = () => {
  // 100+ 状态
  // 50+ 函数
  // ...
};
```

### ❌ 错误 2：未拆分复用组件

```tsx
// ❌ 禁止：重复的表单代码
const Page1 = () => (
  <Form>
    <Form.Item name="name" label="名称"><Input /></Form.Item>
    <Form.Item name="email" label="邮箱"><Input /></Form.Item>
    {/* ... */}
  </Form>
);

// ✅ 正确：抽取通用组件
const UserForm: React.FC = ({ initialValues, onSubmit }) => (
  <Form initialValues={initialValues} onFinish={onSubmit}>
    <Form.Item name="name" label="名称"><Input /></Form.Item>
    <Form.Item name="email" label="邮箱"><Input /></Form.Item>
  </Form>
);
```

### ❌ 错误 3：组件直接调用 API

```tsx
// ❌ 禁止：组件内直接请求
const TaskForm: React.FC = () => {
  const [loading, setLoading] = useState(false);

  const handleSubmit = async () => {
    await request('/api/tasks', { method: 'POST' });  // 直接请求
  };
};
```

### ✅ 正确：组件与逻辑分离

```tsx
// 组件只负责 UI
const TaskForm: React.FC<Props> = ({ onSubmit }) => (
  <Form onFinish={onSubmit}>
    {/* 表单项 */}
  </Form>
);

// 页面负责数据逻辑
const TaskPage: React.FC = () => {
  const handleSubmit = async (values) => {
    await createTask(values);
    actionRef.current?.reload();
  };

  return <TaskForm onSubmit={handleSubmit} />;
};
```

## 基准页面结构（以 PasswordBook 模块为例）

> **重要更新**：`password-book` 模块已重构为**极简单文件模式**，详见 [06-页面开发标准.md](./06-页面开发标准.md)

本项目将 `src/pages/password-book` 模块定义为**前端页面的开发标准参考**。

### 最新目录结构（推荐）

```
src/pages/password-book/
├── index.tsx              # 主页面（所有代码，≤200行）
└── (无其他文件)            # API 内联，组件内联
```

### 核心原则

1. **单文件优先**：
   - 功能简单的页面，合并到单个 `index.tsx` 文件中
   - API 直接内联，不单独创建 `services/{module}/api.ts`
   - 子组件直接定义为内联函数 `React.FC<Props>`

2. **使用 ProTable + ModalForm**：
   - 列表使用 `@ant-design/pro-table` 的 `ProTable`
   - 表单使用 `@ant-design/pro-form` 的 `ModalForm`
   - 内置分页、搜索、排序支持

3. **状态管理简化**：
   - 使用 `set()` 辅助函数简化状态更新
   - 编辑表单使用 `key` 强制重新挂载
