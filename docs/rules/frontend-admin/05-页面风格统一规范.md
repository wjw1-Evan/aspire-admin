# 页面风格统一规范

本规范补充接口交互相关内容，强调所有页面风格、数据流、接口调用、分页、权限、错误处理等与后端标准一致。

> **[强制]** 所有列表/管理页面必须遵循以下统一风格，并与后端 API 结构、权限、分页、错误处理等保持一致。

## 1. 页面容器

统一使用 `@ant-design/pro-components` 的 `PageContainer`，样式如下：

```tsx
import { PageContainer } from '@ant-design/pro-components';

<PageContainer>
  {/* 页面内容 */}
</PageContainer>
```

- 统一使用 `@ant-design/pro-components` 的 PageContainer
- **禁止**自定义 padding 或 margin 覆盖默认样式

## 2. 统计卡片区域

必须使用 StatCard 组件，示例：

```tsx
import { StatCard } from '@/components';
import { Card, Row, Col } from 'antd';

<Card style={{ marginBottom: 16, borderRadius: 12 }}>
  <Row gutter={[12, 12]}>
    <Col xs={24} sm={12} md={6}>
      <StatCard
        title="总任务数"
        value={statistics.totalTasks}
        icon={<TaskIcon />}
        color="#1890ff"
      />
    </Col>
    <Col xs={24} sm={12} md={6}>
      <StatCard
        title="进行中"
        value={statistics.inProgressTasks}
        icon={<ClockCircleIcon />}
        color="#52c41a"
      />
    </Col>
  </Row>
</Card>
```

- 图标在左，数值和标题在右垂直排列
- 卡片内边距：`10px 12px`
- 图标尺寸：`20px`
- 数值字号：`20px`
- 标题字号：`12px`

## 3. 响应式断点

- 移动端 `xs={24}`：单列
- 平板 `sm={12}`：两列
- 桌面 `md={6}`：四列

## 4. 接口交互与数据流

- 所有页面的数据流、接口调用、分页、权限、错误处理等，必须与后端 API 结构、权限标识、分页参数、ApiResponse 结构保持一致。
- 统一 message 提示，401/403/500 由 request 拦截器处理。

## 5. 变更与维护建议

- 页面风格如需扩展，优先遵循本规范和后端标准。
- 发现实现与本规范或后端标准不符时，优先以本规范为准，及时修订文档与代码。

## 数据表格

### 统一使用原生 Table

- 所有列表页面使用 Ant Design 原生 `Table` 组件
- 通过 `onChange` 处理分页、排序、筛选
- 操作列固定 150px，固定在右侧

### 列配置规范

```tsx
import type { ColumnsType } from 'antd/es/table';

const columns: ColumnsType<TaskDto> = [
  {
    title: '任务名称',
    dataIndex: 'taskName',
    width: 200,
    ellipsis: true,
    sorter: true,
  },
  {
    title: '状态',
    dataIndex: 'status',
    width: 100,
    render: (status: number) => <Tag color="blue">{status === 0 ? '待分配' : '已分配'}</Tag>,
  },
  {
    title: '优先级',
    dataIndex: 'priority',
    width: 80,
  },
  {
    title: '负责人',
    dataIndex: 'assignedToName',
    width: 120,
  },
  {
    title: '截止日期',
    dataIndex: 'plannedEndTime',
    width: 120,
  },
  {
    title: '操作',
    key: 'action',
    width: 150,
    fixed: 'right',  // 固定在右侧
  },
];
```

### 操作列规范

```tsx
{
  title: '操作',
  key: 'action',
  width: 150,
  fixed: 'right',
  render: (_, record) => (
    <Space size="small">
      <Button type="link" size="small" onClick={() => handleView(record)}>
        查看
      </Button>
      <Button type="link" size="small" onClick={() => handleEdit(record)}>
        编辑
      </Button>
      <Popconfirm title="确认删除？" onConfirm={() => handleDelete(record.id)}>
        <Button type="link" size="small" danger>
          删除
        </Button>
      </Popconfirm>
    </Space>
  ),
}
```

### 分页与排序处理

```tsx
<Table
  dataSource={data}
  columns={columns}
  rowKey="id"
  loading={loading}
  onChange={(pagination, _filters, sorter) => {
    searchParamsRef.current = {
      ...searchParamsRef.current,
      page: pagination.current,
      pageSize: pagination.pageSize,
      sortBy: sorter.field as string,
      sortOrder: sorter.order === 'ascend' ? 'asc' : sorter.order === 'descend' ? 'desc' : undefined,
    };
    fetchData();
  }}
  pagination={{
    current: pagination.page,
    pageSize: pagination.pageSize,
    total: pagination.total
  }}
/>
```

## 样式规范速查

### 间距规范

| 元素 | 间距 |
|------|------|
| 区块内边距 | 16px |
| 分区间距 | 12px |
| 表单项上下 | 12px |
| 按钮组间距 | 8px |
| 列表顶部/底部留白 | 12px |

### 文字规范

| 元素 | 字号/字重 |
|------|------------|
| 一级标题 | 24px / 600 |
| 卡片标题 | 16px / 600 |
| 正文 | 14px / 400 |
| 次要文字 | 12px / 400 |

### 颜色规范

- 主色：使用 AntD token `colorPrimary`
- **禁止**使用自定义全局色板
- 状态色：统一使用 AntD 语义色 `success` / `info` / `warning` / `error`

### 组件规范

| 组件 | 规范 |
|------|------|
| 卡片 | `<Card bordered={false}>` |
| 表单按钮 | 固定在底部吸底区或抽屉底部 |
| 删除按钮 | 必须使用 `Popconfirm` 确认 |
| 图标 | 使用 `@ant-design/icons`，尺寸 14/16px |

## 常见错误示例

### ❌ 错误 1：自定义全局样式

```tsx
// ❌ 禁止：自定义全局 CSS class
<div className="my-custom-class">

// ✅ 正确：使用 AntD 组件和内联样式
<div style={{ padding: 16 }}>
```

### ❌ 错误 2：表格未固定操作列

```tsx
// ❌ 禁止
{
  title: '操作',
  // 缺少 fixed: 'right'
}

// ✅ 正确
{
  title: '操作',
  fixed: 'right',
  width: 150,
}
```

### ❌ 错误 3：删除未使用 Popconfirm

```tsx
// ❌ 禁止
<Button danger onClick={handleDelete}>删除</Button>

// ✅ 正确
<Popconfirm title="确认删除？" onConfirm={handleDelete}>
  <Button danger>删除</Button>
</Popconfirm>
```

### ❌ 错误 4：自定义主色

```tsx
// ❌ 禁止：自定义主色
<div style={{ color: '#ff0000' }}>

// ✅ 正确：使用 AntD token
<div style={{ color: 'var(--ant-primary-color)' }}>
```
