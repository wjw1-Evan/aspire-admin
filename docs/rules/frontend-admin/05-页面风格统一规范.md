# 页面风格统一规范

本规范补充接口交互相关内容，强调所有页面风格、数据流、接口调用、分页、权限、错误处理等与后端标准一致。

> **[强制]** 所有列表/管理页面必须遵循以下统一风格，并与后端 API 结构、权限、分页、错误处理等保持一致。

## 1. 页面容器

统一使用 `@ant-design/pro-components` 的 `PageContainer`，样式如下：

```tsx
import { PageContainer } from '@ant-design/pro-components';

<PageContainer>
  {/* 页面内容 */}
</PageContainer>
```

- 统一使用 `@ant-design/pro-components` 的 PageContainer
- **禁止**自定义 padding 或 margin 覆盖默认样式

## 2. 统计卡片区域

### 推荐方案：使用 ProCard gutter 布局

优先使用 `ProCard` 的 `gutter` 属性实现统计卡片布局，更简洁：

```tsx
import { ProCard } from '@ant-design/pro-components';

<ProCard gutter={16} style={{ marginBottom: 16 }}>
  <ProCard colSpan={{ xs: 24, sm: 12, md: 6 }}>
    <div style={{ fontSize: 24, fontWeight: 'bold', color: '#1890ff' }}>100</div>
    <div style={{ color: '#8c8c8c', fontSize: 12 }}>总用户数</div>
  </ProCard>
  <ProCard colSpan={{ xs: 24, sm: 12, md: 6 }}>
    <div style={{ fontSize: 24, fontWeight: 'bold', color: '#52c41a' }}>80</div>
    <div style={{ color: '#8c8c8c', fontSize: 12 }}>活跃用户</div>
  </ProCard>
  <ProCard colSpan={{ xs: 24, sm: 12, md: 6 }}>
    <div style={{ fontSize: 24, fontWeight: 'bold', color: '#faad14' }}>10</div>
    <div style={{ color: '#8c8c8c', fontSize: 12 }}>管理员</div>
  </ProCard>
  <ProCard colSpan={{ xs: 24, sm: 12, md: 6 }}>
    <div style={{ fontSize: 24, fontWeight: 'bold', color: '#722ed1' }}>5</div>
    <div style={{ color: '#8c8c8c', fontSize: 12 }}>新增用户</div>
  </ProCard>
</ProCard>
```

### 备选方案：StatCard 组件

如需使用 StatCard 组件，配合 Card + Row/Col：

```tsx
import { StatCard } from '@/components';
import { Card, Row, Col } from 'antd';

<Card style={{ marginBottom: 16 }}>
  <Row gutter={[12, 12]}>
    <Col xs={24} sm={12} md={6}>
      <StatCard title="总任务数" value={statistics.totalTasks} icon={<TaskIcon />} color="#1890ff" />
    </Col>
  </Row>
</Card>
```

## 3. 数据表格 (ProTable)

### 基本配置

所有列表页面统一使用 `ProTable` 组件：

```tsx
import { ProTable } from '@ant-design/pro-table';
import { Input } from 'antd';
import { SearchOutlined, ReloadOutlined, PlusOutlined } from '@ant-design/icons';

<ProTable
  actionRef={actionRef}
  request={async (params) => {
    const { current, pageSize } = params;
    const res = await api.list({ page: current, pageSize, search: state.search });
    return { data: res.data?.queryable || [], total: res.data?.rowCount || 0, success: res.success };
  }}
  columns={columns}
  rowKey="id"
  search={false}
  scroll={{ x: 'max-content' }}
  toolBarRender={() => [
    <Input.Search
      key="search"
      placeholder="搜索..."
      allowClear
      value={state.search}
      onChange={(e) => set({ search: e.target.value })}
      onSearch={(value) => { set({ search: value }); actionRef.current?.reload(); }}
      style={{ width: 260, marginRight: 8 }}
      prefix={<SearchOutlined />}
    />,
    <Button key="refresh" icon={<ReloadOutlined />} onClick={handleRefresh}>刷新</Button>,
    <Button key="create" type="primary" icon={<PlusOutlined />} onClick={() => handleOpenModal(null)}>
      新建
    </Button>,
  ]}
/>
```

### 关键配置说明

| 属性 | 值 | 说明 |
|------|-----|------|
| `search={false}` | false | 禁用 ProTable 内置搜索表单，使用自定义搜索框 |
| `scroll={{ x: 'max-content' }}` | x: 'max-content' | **必须添加**，让列宽自动适应内容 |
| `toolBarRender` | function | **必须包含**搜索框、刷新、新建等操作按钮 |

### 搜索框统一规范

搜索框必须：
1. 使用 `Input.Search` 组件
2. 添加 `prefix={<SearchOutlined />}` 图标
3. 使用受控模式 `value` + `onChange`
4. 搜索时触发 `actionRef.current?.reload()`
5. 宽度固定 `260px`，右边距 `8px`

```tsx
<Input.Search
  placeholder="搜索..."
  allowClear
  value={state.search}
  onChange={(e) => set({ search: e.target.value })}
  onSearch={(value) => { set({ search: value }); actionRef.current?.reload(); }}
  style={{ width: 260, marginRight: 8 }}
  prefix={<SearchOutlined />}
/>
```

### 列配置规范

```tsx
const columns: ProColumns<DataType>[] = [
  {
    title: '名称',
    dataIndex: 'name',
    ellipsis: true,
    sorter: true,
  },
  {
    title: '状态',
    dataIndex: 'status',
    render: (_, record) => <Tag color="blue">{record.statusName}</Tag>,
  },
  // 操作列固定在右侧
  {
    title: '操作',
    key: 'action',
    width: 150,
    fixed: 'right',
    valueType: 'option',
    render: (_, record) => (
      <Space>
        <Button type="link" size="small" onClick={() => handleEdit(record)}>编辑</Button>
        <Button type="link" size="small" danger onClick={() => handleDelete(record)}>删除</Button>
      </Space>
    ),
  },
];
```

### 操作列规范

```tsx
{
  title: '操作',
  key: 'action',
  width: 150,
  fixed: 'right',  // 仅当最后一列为操作按钮时才固定
  valueType: 'option',
  render: (_, record) => (
    <Space size="small">
      <Button type="link" size="small" icon={<EditOutlined />} onClick={() => handleEdit(record)}>
        编辑
      </Button>
      <Popconfirm title="确认删除？" onConfirm={() => handleDelete(record.id)}>
        <Button type="link" size="small" danger icon={<DeleteOutlined />}>
          删除
        </Button>
      </Popconfirm>
    </Space>
  ),
}
```

## 4. 详情展示 (ProDescriptions)

使用 `ProDescriptions` 的子组件方式，**不要**使用 `items` 属性：

```tsx
import { ProDescriptions } from '@ant-design/pro-components';

<ProDescriptions column={2} bordered size="small">
  <ProDescriptions.Item label="名称">{data.name}</ProDescriptions.Item>
  <ProDescriptions.Item label="状态">
    <Tag color="blue">{data.status}</Tag>
  </ProDescriptions.Item>
  <ProDescriptions.Item label="描述" span={2}>
    {data.description || '-'}
  </ProDescriptions.Item>
  <ProDescriptions.Item label="创建时间">
    {dayjs(data.createdAt).format('YYYY-MM-DD HH:mm:ss')}
  </ProDescriptions.Item>
</ProDescriptions>
```

## 5. 响应式断点

- 移动端 `xs={24}`：单列
- 平板 `sm={12}`：两列
- 桌面 `md={6}`：四列

## 6. 接口交互与数据流

- 所有页面的数据流、接口调用、分页、权限、错误处理等，必须与后端 API 结构、权限标识、分页参数、ApiResponse 结构保持一致。
- 统一 message 提示，401/403/500 由 request 拦截器处理。

## 7. API 路由规范

前端调用 API 时必须与后端 Controller 路由完全匹配：

| 后端 Controller | 后端路由 | 前端调用示例 |
|----------------|----------|-------------|
| `/api/park/contracts` | POST /api/park/contracts/list | `{ method: 'POST', data: params }` |
| `/api/storage-quota` | GET /api/storage-quota/list | `{ params }` |
| `/api/park/contracts/payments` | POST /api/park/contracts/payments | `{ method: 'POST', data }` |

> **注意**：分页列表接口通常为 POST，使用 `{ data: params }` 而非 `{ params }`

## 样式规范速查

### 间距规范

| 元素 | 间距 |
|------|------|
| 区块内边距 | 16px |
| 分区间距 | 12px |
| 表单项上下 | 12px |
| 按钮组间距 | 8px |
| 列表顶部/底部留白 | 12px |

### 文字规范

| 元素 | 字号/字重 |
|------|------------|
| 一级标题 | 24px / 600 |
| 卡片标题 | 16px / 600 |
| 正文 | 14px / 400 |
| 次要文字 | 12px / 400 |

### 颜色规范

- 主色：使用 AntD token `colorPrimary`
- **禁止**使用自定义全局色板
- 状态色：统一使用 AntD 语义色 `success` / `info` / `warning` / `error`

### 组件规范

| 组件 | 规范 |
|------|------|
| 卡片 | `<Card bordered={false}>` |
| 表格 | ProTable + scroll={{ x: 'max-content' }} |
| 表单按钮 | 固定在底部吸底区或抽屉底部 |
| 删除按钮 | 必须使用 `Popconfirm` 确认 |
| 图标 | 使用 `@ant-design/icons`，尺寸 14/16px |

## 常见错误示例

### ❌ 错误 1：自定义全局样式

```tsx
// ❌ 禁止：自定义全局 CSS class
<div className="my-custom-class">

// ✅ 正确：使用 AntD 组件和内联样式
<div style={{ padding: 16 }}>
```

### ❌ 错误 2：ProTable 未添加列宽自适应

```tsx
// ❌ 禁止
<ProTable ... />

// ✅ 正确
<ProTable ... scroll={{ x: 'max-content' }} />
```

### ❌ 错误 3：搜索框未使用受控模式

```tsx
// ❌ 禁止
<Input.Search placeholder="搜索..." />

// ✅ 正确
<Input.Search
  placeholder="搜索..."
  value={state.search}
  onChange={(e) => set({ search: e.target.value })}
  onSearch={(value) => { set({ search: value }); actionRef.current?.reload(); }}
/>
```

### ❌ 错误 4：ProDescriptions 使用 items 属性

```tsx
// ❌ 禁止
<ProDescriptions items={ [...]} />

// ✅ 正确
<ProDescriptions column={2} bordered>
  <ProDescriptions.Item label="名称">{data.name}</ProDescriptions.Item>
</ProDescriptions>
```

### ❌ 错误 5：删除未使用 Popconfirm

```tsx
// ❌ 禁止
<Button danger onClick={handleDelete}>删除</Button>

// ✅ 正确
<Popconfirm title="确认删除？" onConfirm={handleDelete}>
  <Button danger>删除</Button>
</Popconfirm>
```

### ❌ 错误 6：自定义主色

```tsx
// ❌ 禁止：自定义主色
<div style={{ color: '#ff0000' }}>

// ✅ 正确：使用 AntD token
<div style={{ color: 'var(--ant-primary-color)' }}>
```

### ❌ 错误 7：非操作列设置固定

```tsx
// ❌ 禁止：非操作列设置 fixed
{ title: '时间', dataIndex: 'createdAt', width: 160, fixed: 'right' }

// ✅ 正确：仅操作列设置固定
{ title: '操作', key: 'action', width: 150, fixed: 'right' }
```

### ❌ 错误 8：分页接口使用 GET

```tsx
// ❌ 禁止：分页列表使用 GET
request('/api/list', { params: { page: 1, pageSize: 10 } })

// ✅ 正确：分页列表使用 POST + data
request('/api/list', { method: 'POST', data: { page: 1, pageSize: 10 } })
```
