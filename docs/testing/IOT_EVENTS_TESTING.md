# IoT 事件告警模块测试指南

## 测试环境准备

### 1. 启动后端服务
```bash
cd Platform.AppHost
dotnet run
```

确保以下服务正常运行：
- API Service (http://localhost:5000)
- MongoDB 数据库
- Redis 缓存

### 2. 启动前端应用
```bash
cd Platform.Admin
npm run dev
```

访问 http://localhost:8000/welcome

## 功能测试

### 测试 1：模块显示
**预期结果**：
- 欢迎页面右侧列显示"物联网事件告警"卡片
- 卡片位置在"审批概览"下方，"系统资源监控"上方

**测试步骤**：
1. 登录系统
2. 访问欢迎页面
3. 检查右侧列是否显示 IoT 事件告警模块

### 测试 2：权限控制
**预期结果**：
- 有 IoT 平台访问权限的用户能看到模块
- 无权限的用户看不到模块

**测试步骤**：
1. 使用有 IoT 权限的账户登录
2. 验证模块显示
3. 切换到无 IoT 权限的账户
4. 验证模块隐藏

### 测试 3：事件显示
**预期结果**：
- 显示最近 5 条未处理事件
- 事件按发生时间倒序排列
- 显示事件级别、类型、描述、设备 ID、发生时间

**测试步骤**：
1. 在数据库中插入测试事件数据
2. 刷新欢迎页面
3. 验证事件列表显示正确

### 测试 4：未处理事件计数
**预期结果**：
- 顶部 Badge 显示未处理事件总数
- 当有未处理事件时显示警告提示

**测试步骤**：
1. 查看 Badge 显示的数字
2. 在数据库中修改事件状态为已处理
3. 刷新页面，验证 Badge 数字减少

### 测试 5：自动轮询
**预期结果**：
- 每 30 秒自动刷新一次
- 页面不可见时停止轮询
- 页面恢复可见时立即刷新

**测试步骤**：
1. 打开浏览器开发者工具（F12）
2. 切换到 Network 标签
3. 观察 API 请求频率
4. 切换浏览器标签页（页面不可见）
5. 验证请求停止
6. 切换回欢迎页面
7. 验证立即发起新请求

### 测试 6：时间格式化
**预期结果**：
- 刚发生的事件显示"刚刚"
- 1 分钟前的事件显示"1分钟前"
- 1 小时前的事件显示"1小时前"
- 1 天前的事件显示"1天前"
- 7 天前的事件显示具体日期

**测试步骤**：
1. 创建不同时间的测试事件
2. 验证时间显示格式

### 测试 7：事件类型显示

1. 创建不同类型的测试事件
2. 验证类型翻译正确

### 测试 8：事件级别颜色

- Critical → 红色

### 测试 9：快速导航

## 常见问题

### Q: 模块不显示？
A: 检查以下几点：
1. 用户是否有 IoT 平台访问权限
2. 后端 API 是否正常运行
3. 浏览器控制台是否有错误信息

### Q: 事件不更新？
A: 检查以下几点：
1. 轮询是否正常工作（检查 Network 标签）
2. 数据库中是否有新事件
3. 页面是否可见

### Q: 时间显示不正确？
A: 检查以下几点：
1. 服务器时间是否正确
2. 浏览器时区设置是否正确
3. 事件的 occurredAt 字段是否正确
